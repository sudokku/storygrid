  <action>
In `tailwind.config.js`, add `animation` and `keyframes` keys inside `theme.extend`, after the existing `fontFamily` entry:

```javascript
animation: {
  'cell-wobble': 'cell-wobble 150ms ease-in-out',
  'drop-flash': 'drop-flash 700ms ease-out',
},
keyframes: {
  'cell-wobble': {
    '0%':   { transform: 'rotate(0deg)' },
    '25%':  { transform: 'rotate(1.5deg)' },
    '75%':  { transform: 'rotate(-1.5deg)' },
    '100%': { transform: 'rotate(0deg)' },
  },
  'drop-flash': {
    '0%':   { boxShadow: '0 0 0 3px var(--primary)' },
    '100%': { boxShadow: 'none' },
  },
},
```

Use `var(--primary)` for drop-flash (not `var(--ring)`): `--primary` is `oklch(0.205 0 0)` in light / `oklch(0.922 0 0)` in dark — higher contrast on the dark canvas cells than `--ring` (`oklch(0.708 0 0)` medium gray).

Do NOT modify any existing keys (`colors`, `borderRadius`, `fontFamily`, `plugins`).

The `keyframes` key in `theme.extend` causes Tailwind to generate the `@keyframes` blocks in the output CSS. The `animation` key generates the `animate-*` utility classes. Both keys are standard Tailwind v3 config patterns.
  </action>

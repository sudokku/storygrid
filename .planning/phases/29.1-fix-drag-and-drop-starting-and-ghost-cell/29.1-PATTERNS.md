**Core modifier pattern — ghost scale + offset computation** (lines 32–71):
```typescript
export const grabOffsetModifier: Modifier = ({
  transform,
  draggingNodeRect,
}) => {
  if (!draggingNodeRect) return transform;
  const { pointerDownX, pointerDownY, sourceW, sourceH } = useDragStore.getState();
  if (pointerDownX === 0 && pointerDownY === 0) return transform;

  const offsetX = pointerDownX - draggingNodeRect.left;
  const offsetY = pointerDownY - draggingNodeRect.top;

  const CAP = 200;
  const ghostScale =
    sourceW > 0 && sourceH > 0 ? Math.min(CAP / sourceW, CAP / sourceH, 1) : 1;
  const ghostW = sourceW * ghostScale;
  const ghostH = sourceH * ghostScale;
  const ghostOffsetX =
    draggingNodeRect.width > 0 ? (offsetX / draggingNodeRect.width) * ghostW : offsetX;
  const ghostOffsetY =
    draggingNodeRect.height > 0 ? (offsetY / draggingNodeRect.height) * ghostH : offsetY;

  const centreDeltaX = ghostOffsetX - ghostW / 2;
  const centreDeltaY = ghostOffsetY - ghostH / 2;

  return {
    ...transform,
    x: transform.x + centreDeltaX,
    y: transform.y + centreDeltaY,
  };
};
```

**BUG — ghost `<img>` render pattern** (lines 88–105):
```tsx
{isDragging && ghostUrl ? (
  <img
    src={ghostUrl}
    width={sourceW}
    height={sourceH}
    className="max-w-[var(--ghost-cap)] max-h-[var(--ghost-cap)]"
    style={{
      opacity: 0.2,
      display: 'block',
      pointerEvents: 'none',
      width: `${sourceW}px`,   // ← BUG: forces full source width, CSS cap then clips independently per axis
      height: `${sourceH}px`,  // ← BUG: both axes capped independently → distorted render ≠ modifier math
    }}
    alt=""
    draggable={false}
  />
) : null}
```

**What D-01/D-02 require (fix target):**

The modifier pre-computes `ghostW` and `ghostH` with uniform aspect-ratio scale (`Math.min(CAP/sourceW, CAP/sourceH, 1)`). The `<img>` must render at exactly those dimensions. The current inline `width`/`height` forces the full source size and lets CSS `max-w`/`max-h` clip each axis independently, which can distort the aspect ratio and mismatch the modifier.

**Correct fix — set both axes to the pre-computed capped values:**
- Remove `className="max-w-[var(--ghost-cap)] max-h-[var(--ghost-cap)]"` (CSS cap no longer needed)
- Set `style={{ width: ghostW, height: ghostH, ... }}` where `ghostW`/`ghostH` are computed with the same `Math.min(CAP/sourceW, CAP/sourceH, 1)` formula used in the modifier
- The component must read `sourceW` and `sourceH` from `useDragStore` (already done at lines 75–76) and perform the same cap calculation locally

**Pattern for local ghost dimension computation in the component:**
```tsx
const CAP = 200;
const ghostScale =
  sourceW > 0 && sourceH > 0 ? Math.min(CAP / sourceW, CAP / sourceH, 1) : 1;
const ghostW = sourceW * ghostScale;
const ghostH = sourceH * ghostScale;
```

This mirrors the modifier's own computation (lines 52–56) and must be kept in sync.

**Drop zone pointer computation — audit target D-05/D-06** (lines 92–103):
```typescript
const handleDragMove = useCallback(({ over, activatorEvent, delta }: DragMoveEvent) => {
  if (!over) {
    useDragStore.getState().setOver(null, null);
    return;
  }
  const rect = over.rect as unknown as DOMRect;
  const ev = activatorEvent as PointerEvent;
  const pointer = { x: ev.clientX + delta.x, y: ev.clientY + delta.y };
  const zone = computeDropZone(rect, pointer);
  useDragStore.getState().setOver(String(over.id), zone);
}, []);
```

Audit verdict (code-read only): `pointer` is derived from `activatorEvent.clientX/Y + delta.x/Y` — this correctly tracks the live pointer position in viewport space, independent of any ghost visual. No fix required per D-05.

## v1.2 Requirements — Final Status

### Effects & Filters (per-cell, non-destructive)

- [x] **EFF-01**: User can apply a named preset filter (B&W, Sepia, Vivid, Fade, Warm, Cool) to any leaf cell from a preset carousel in the sidebar ✅ Phase 11
- [x] **EFF-02**: User can adjust brightness on the selected cell via a slider (-100 to +100, default 0) ✅ Phase 11
- [x] **EFF-03**: User can adjust contrast on the selected cell via a slider (-100 to +100, default 0) ✅ Phase 11
- [x] **EFF-04**: User can adjust saturation on the selected cell via a slider (-100 to +100, default 0) ✅ Phase 11
- [x] **EFF-05**: User can adjust blur on the selected cell via a slider (0 to 20px, default 0) ✅ Phase 11
- [x] **EFF-06**: User can reset all effects on a cell to defaults via a single "Reset" button ✅ Phase 11
- [x] **EFF-07**: Effects are non-destructive — the original media is never modified; effect parameters are stored on the leaf node ✅ Phase 11
- [x] **EFF-08**: Effects apply identically in the live preview canvas, PNG export, and MP4 video export (WYSIWYG) ✅ Phase 11
- [x] **EFF-09**: Effects survive undo/redo; slider drags commit a single history snapshot on pointerup (not per pointermove) ✅ Phase 11
- [x] **EFF-10**: Applying a preset updates the underlying slider values so the user can fine-tune after selecting a preset ✅ Phase 11

### Text & Sticker Overlay Layer (global, above the grid)

- [x] **OVL-01**: User can add a text overlay to the canvas via a "Add Text" button; new text spawns centered on the canvas with placeholder text ✅ Phase 13
- [x] **OVL-02**: User can edit the text content of a selected text overlay via an inline editor or sidebar input ✅ Phase 13
- [x] **OVL-03**: User can change the font family of a selected text overlay (minimum 3 choices: Geist, serif display, script) ✅ Phase 13
- [x] **OVL-04**: User can change the font size of a selected text overlay via a slider (16–256px) ✅ Phase 13
- [x] **OVL-05**: User can change the text color of a selected text overlay via a color picker ✅ Phase 13
- [x] **OVL-06**: User can change the font weight of a selected text overlay (regular / bold) ✅ Phase 13
- [x] **OVL-07**: User can change the text alignment of a selected text overlay (left / center / right) ✅ Phase 13
- [x] **OVL-08**: User can add an emoji sticker via an emoji picker panel; emoji spawns centered on the canvas ✅ Phase 13
- [x] **OVL-09**: User can upload a custom PNG or SVG image sticker via a file picker; SVG content is sanitized via DOMPurify before use ✅ Phase 13
- [x] **OVL-10**: User can drag any overlay (text, emoji, image sticker) freely across the 1080×1920 canvas ✅ Phase 13
- [x] **OVL-11**: User can resize any selected overlay via a corner handle (proportional scaling) ✅ Phase 13
- [x] **OVL-12**: User can rotate any selected overlay via a top rotation handle ✅ Phase 13
- [x] **OVL-13**: User can delete a selected overlay via the Delete key or a trash button ✅ Phase 13
- [x] **OVL-14**: User can reorder overlays front-to-back via "Bring Forward" / "Send Backward" actions ✅ Phase 13
- [x] **OVL-15**: Overlay selection is mutually exclusive with cell selection — clicking one clears the other ✅ Phase 13
- [x] **OVL-16**: Overlays render identically in the live preview, PNG export, and MP4 video export, at the correct 1080×1920 coordinates ✅ Phase 13
- [x] **OVL-17**: Sticker image data is stored in a separate `stickerRegistry` (mirroring `mediaRegistry`) — never inlined into the grid tree or history snapshots ✅ Phase 13

### Per-Cell Audio Toggle (video export)

- [x] **AUD-01**: Each video cell has an audio-enabled boolean field on its leaf node, defaulting to `true` ✅ Phase 12
- [x] **AUD-02**: Each video cell shows a speaker icon (unmuted/muted) in the portal ActionBar ✅ Phase 12
- [x] **AUD-03**: Clicking the speaker icon toggles the cell's audio-enabled state ✅ Phase 12
- [x] **AUD-04**: The sidebar video cell panel also exposes the audio toggle ✅ Phase 12
- [x] **AUD-05**: MP4 video export mixes audio from all enabled cells via Web Audio API graph ✅ Phase 12 + 14
- [x] **AUD-06**: If zero cells have audio enabled, exported MP4 contains no audio track ✅ Phase 12
- [x] **AUD-07**: Audio in the exported MP4 is clipped to the total video export duration ✅ Phase 12
- [ ] **AUD-08**: Audio toggle state is persisted in saved projects and `.storygrid` files — *Deferred to v1.3 (depends on PERS block)*
- [x] **AUD-09**: Audio export is Chrome/Firefox only — matches existing video export browser support boundary ✅ Phase 12

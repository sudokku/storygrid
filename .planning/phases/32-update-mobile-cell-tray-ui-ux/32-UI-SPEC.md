## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap inside tray buttons (`gap-0.5` = 2px, use `gap-1` = 4px) |
| sm | 8px | Gap between tray buttons (`gap-2`) |
| md | 16px | Horizontal padding inside tray container (`px-4`) |
| lg | 24px | — |
| xl | 32px | — |
| 2xl | 48px | Tab strip height (target, replaces 60px) |
| 3xl | 64px | — |

Exceptions:
- Touch targets: minimum 44×44px on all tray buttons (`min-w-[44px] min-h-[44px]`), per Apple HIG. This is not a spacing token — it is a touch-target floor.
- Tray `bottom` offset: 48px (matches new tab strip height, down from 60px). Updated alongside `SNAP_TRANSLATE.collapsed`.
- Header outer height: 44px (`h-11`). Reduced from 48px (`h-12`). Buttons remain 44px tall — outer element matches the tallest button exactly, no overflow. (Source: RESEARCH.md open question resolved — `h-10`/40px clips 44px buttons, so `h-11`/44px is the minimum safe value.)

Source: CONTEXT.md D-06, D-11, D-12; RESEARCH.md Pitfall 4, Pitfall 5; UI-INTELLIGENCE.md Touch Target Size

## Typography

The tray introduces one new typographic element: the button label (D-04). All other UI chrome is unchanged.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Button label (tray) | 10px (`text-[10px]`) | 400 regular | 1 (`leading-none`) | Label text below each tray icon |
| Body / panel text | 14px (`text-sm`) | 400 regular | 1.5 | Sheet content, sidebar |
| UI label | 12px (`text-xs`) | 500 medium | 1.25 | Tab strip tabs, section headers |
| Heading | 16px (`text-base`) | 600 semibold | 1.2 | Sheet panel headings |

Weights in use: 400 regular + 600 semibold (2 weights max).

Label truncation strategy: tray button labels are single words (max 8 chars: "Upload", "Split H", "Split V", "Fit", "Clear", "Effects", "Audio"). No truncation needed. If a future label exceeds button width, use `truncate` with `max-w-[44px]` on the `<span>`.

Source: CONTEXT.md D-04; RESEARCH.md Pattern 3; existing `BTN_CLASS` in `MobileCellTray.tsx`

## Interaction Contract

### Tray Button Layout (D-04, D-06)

Each button stacks icon + label vertically:

```
┌─────────────────┐
│   [icon 20px]   │
│   label text    │  ← 10px, text-white, leading-none
└─────────────────┘
  min-w/h: 44px
```

Updated `BTN_CLASS`:
```
min-w-[44px] min-h-[44px] flex flex-col items-center justify-center gap-1 rounded hover:bg-white/10 transition-colors focus-visible:ring-2 focus-visible:ring-[var(--ring)] focus-visible:outline-none
```

Tray container: drop `justify-center`, add `overflow-x-auto`, hide scrollbar chrome:
```
mx-4 flex items-center gap-2 px-4 py-1 bg-black/70 backdrop-blur-sm rounded-md overflow-x-auto [&::-webkit-scrollbar]:hidden
```

All buttons get `flex-shrink-0` to prevent collapse in scroll context.

### Button Order (D-03)

Left → right: **Upload → Split H → Split V → Fit → Clear (conditional) → Effects → Audio (conditional)**

Visibility rules:
- **Upload**: always shown when cell is selected
- **Split H**: always shown when cell is selected
- **Split V**: always shown when cell is selected
- **Fit**: always shown when cell is selected (icon swaps: `Minimize2` ↔ `Maximize2`)
- **Clear**: shown only when `hasMedia === true` (`{hasMedia && <button>}`)
- **Effects**: always shown when cell is selected (calls `setSheetSnapState('full')`)
- **Audio**: shown only when `isVideo === true` (condition: `mediaTypeMap[mediaId] === 'video'`); additionally, `disabled` when `!hasAudioTrack`; icon swaps: `Volume2` ↔ `VolumeX`

Note: Audio button mirrors the full ActionBar guard (`isVideo && hasAudioTrack` for interactive, `isVideo && !hasAudioTrack` for disabled). This resolves RESEARCH.md open question 1 in favor of consistency with ActionBar.

### Scroll Fade Edges (D-06, Claude's Discretion)

Add a CSS `mask-image` fade on the tray container's scroll div to signal scrollability:

```css
mask-image: linear-gradient(to right, transparent 0px, black 12px, black calc(100% - 12px), transparent 100%)
```

Applied as Tailwind arbitrary value on the scroll div wrapper (not the button container directly, to avoid clipping button hit areas).

Decision: add fade edges — the tray will hold 7 buttons on narrow phones (~320px) where 7 × 44px = 308px + 6 × 8px gaps = 356px, which exceeds 320px minus 32px margins (288px available). Fades are necessary to signal hidden buttons.

### Tray Visibility Gate (D-07)

Three conditions hide the tray (opacity 0, pointerEvents none):
1. `isDragging === true` (existing)
2. `!isVisible` i.e. `selectedNodeId === null` (existing)
3. `sheetSnapState === 'full'` (new — Phase 32)

Transition: `opacity 0.3s cubic-bezier(0.32, 0.72, 0, 1)` (existing, unchanged).

Reduced-motion: `motion-reduce:transition-none` class on outer div (existing, unchanged).

### Effects Button Behavior (D-08)

Tap Effects → `setSheetSnapState('full')` → tray hides via D-07 → sheet expands to full. No scroll-to-section needed: the sheet content already shows `EffectsPanel` for selected cells. User returns to tray by tapping the sheet chevron (collapses sheet → `setSheetSnapState('collapsed')` → tray re-appears).

### Sheet Auto-Expand Removal (D-09)

Cell tap no longer auto-expands the sheet. The `useEffect` block in `MobileSheet.tsx` that calls `setSheetSnapState('full')` when `selectedNodeId` transitions from `null → non-null` is deleted.

The overlay-selection `useEffect` (identical pattern but for `selectedOverlayId`) is kept unchanged (D-10).

### Header Compaction (D-11)

Mobile branch of `Toolbar.tsx`: change outer `<header>` from `h-12` (48px) to `h-11` (44px). Button sizes remain `w-11 h-11` (44px) — the outer element exactly contains them with zero vertical padding. No button height change.

### SNAP_TRANSLATE Update (D-12, D-13)

New tab strip height: **48px** (down from 60px).
New header height: **44px** (down from 48px).

Updated constants in `MobileSheet.tsx`:

```tsx
const SNAP_TRANSLATE: Record<SheetSnapState, string> = {
  full: 'translateY(max(calc(env(safe-area-inset-top) + 44px), 60px))',  // was 56px → 44px
  collapsed: 'translateY(calc(100% - 48px))',                              // was 60px → 48px
};
```

Content scroll area height calc (line ~86):
```
height: 'calc(100dvh - 48px - max(calc(env(safe-area-inset-top) + 44px), 60px))'
```

Also update: `style={{ height: 60 }}` on the tab strip div → `style={{ height: 48 }}`.

Also update: `MobileCellTray.tsx` line 106 `bottom: '60px'` → `bottom: '48px'`.

All four occurrences of the old heights must be updated together or the layout breaks (RESEARCH.md Pitfall 3, Pitfall 5).

## Accessibility Contract

- All tray buttons: `aria-label` required (no text label alone — the label is 10px and insufficient for AT without aria support). Each button carries an explicit `aria-label` as listed above.
- Audio button disabled state: `disabled` attribute when `isVideo && !hasAudioTrack`. `aria-label="No audio track"` on the disabled button.
- Tab order: buttons appear in DOM order matching visual order (Upload → Split H → Split V → Fit → Clear → Effects → Audio). No `tabIndex` reordering needed.
- Focus rings: `focus-visible:ring-2 focus-visible:ring-[var(--ring)] focus-visible:outline-none` already in `BTN_CLASS` — preserved.

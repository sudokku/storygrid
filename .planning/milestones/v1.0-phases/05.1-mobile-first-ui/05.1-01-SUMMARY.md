# Phase 05.1 Plan 01: Responsive Foundation — Mobile Layout and Bottom Sheet Summary

Mobile-first responsive foundation: useMediaQuery hook, sheetSnapState in editorStore, MobileSheet bottom sheet with 3 CSS snap states driven by drag gestures, responsive layout switch at 768px breakpoint, and expanded 22px Divider hit areas for touch.

## What Was Built

### Task 1: useMediaQuery + editorStore + Responsive Layout + Divider Touch

**useMediaQuery hook** (`src/hooks/useMediaQuery.ts`): Extracted from Sidebar.tsx where it was defined locally. Now a shared hook importable by Toolbar.tsx and MobileSheet. Uses addEventListener on MediaQueryList with proper cleanup.

**editorStore additions** (`src/store/editorStore.ts`): Added `sheetSnapState: 'collapsed' | 'half' | 'full'` field and `setSheetSnapState` setter. Initial value is `'collapsed'`.

**Sidebar refactor** (`src/Editor/Sidebar.tsx`):
- Removed local `useMediaQuery` definition
- Removed `isTooSmall` and `isNarrow` logic + D-25 desktop notice (the `data-testid="desktop-notice"` div)
- Added `hidden md:flex` CSS classes — sidebar now hidden on mobile via CSS, not JavaScript
- Exported `CanvasSettingsPanel` and `SelectedCellPanel` for use in MobileSheet

**Toolbar refactor** (`src/Editor/Toolbar.tsx`):
- Added `isMobile = useMediaQuery('(max-width: 767px)')` early return
- Mobile branch renders: `<header>` with "StoryGrid" wordmark + Export button (`data-testid="export-button"`)
- Mobile Export button uses same export pipeline as ExportSplitButton (reads state at call time, no stale closures)
- Desktop toolbar JSX unchanged

**CanvasArea** (`src/Editor/CanvasArea.tsx`): Changed `p-8` to `p-0 pt-2 md:p-8`. Canvas fills full viewport width on mobile with minimal top padding.

**EditorShell** (`src/Editor/EditorShell.tsx`): Added `<MobileSheet />` after canvas/sidebar row. Inner flex div gets `pb-[60px] md:pb-0` to reserve space for collapsed sheet handle.

**Divider hit areas** (`src/Grid/Divider.tsx`): Expanded from 8px to 22px (`h-[22px]`/`w-[22px]`). Offset changed from `-4px` to `-11px` to keep centered. Visible line and grab handle repositioned to remain centered within expanded zone.

### Task 2: MobileSheet Component + Foundation Tests

**MobileSheet** (`src/Editor/MobileSheet.tsx`):
- Fixed-position bottom sheet with `md:hidden` (invisible on desktop)
- `SNAP_TRANSLATE` map: `collapsed` = `translateY(calc(100% - 60px))`, `half` = `translateY(60vh)`, `full` = `translateY(0)`
- Drag handle: `touch-none` + `cursor-grab` + `setPointerCapture` for reliable touch input
- Pointer up: computes `dy` vs `DRAG_THRESHOLD = 50` to advance/retreat snap state
- `transition: 'transform 0.3s cubic-bezier(0.32, 0.72, 0, 1)'` disabled during active drag
- Header row: `TemplatesPopover` + Undo + Redo + Clear buttons (44px tap targets)
- `clearGrid` called directly (no `window.confirm()`) per UI-SPEC
- Content area: pan mode → Exit Pan Mode button | selected cell → SelectedCellPanel | default → CanvasSettingsPanel
- `useEffect` auto-expands to `half` when `selectedNodeId` changes to non-null

**Foundation tests** (`src/Editor/__tests__/phase05.1-p01-foundation.test.tsx`): 15 tests covering MobileSheet behavior (snap states, drag gestures, context-sensitive content, pan mode, undo/redo disabled state, D-02 removal assertion).

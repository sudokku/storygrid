## Component Inventory

### Modified Components

**ActionBar.tsx** — sizing changes only

- Button size: change from fixed `w-8 h-8` (32px) to `clamp(28px, 2.2vw, 36px)` applied via inline style or a CSS custom property `--ab-btn-size`.
- Icon size: change from fixed `size={16}` to a computed value `Math.round(abBtnSize * 0.5)` where `abBtnSize` is derived from `canvasScale` and the `clamp()` formula — keeping the icon at 50% of button size.
- The `scale(1/canvasScale)` transform in `LeafNode.tsx` counteracts the canvas zoom so the bar appears physically stable. The `clamp()` values are authored at the logical (unscaled) size; the inverse-scale transform produces stable physical px across all zoom levels (D-05).

**LeafNode.tsx** — two structural changes

1. **Overflow isolation (D-01, D-02, D-03):**
   - The outer cell `div` currently has `overflow-hidden` on the root class. Change this to `overflow-visible`.
   - Wrap the `<canvas>` element (and only the canvas) in a new `<div className="absolute inset-0 overflow-hidden">` to preserve media clipping.
   - Ensure no intermediate wrapper between the cell container and the overflow-visible root reintroduces `overflow-hidden`.

2. **Empty cell placeholder scaling (D-07, D-08):**
   - Replace fixed `size={24}` on `ImageIcon` with a canvas-relative computed size, e.g. `clamp(20px, 1.6vw, 32px)` via inline style `width` and `height`.
   - Replace fixed `text-sm` on the label span with `text-[clamp(10px,0.7vw,14px)]` using Tailwind arbitrary value syntax.
   - Hide the label entirely when the cell's rendered pixel height is below 80px. Use a `ResizeObserver` callback on `divRef` (already present in `LeafNode.tsx`) to set a local boolean state `isTooSmall`. When `isTooSmall` is true, render the label with `className="hidden"`.

**Sidebar.tsx (SelectedCellPanel)** — thumbnail logic change (D-11)

- The existing `<img src={mediaUrl}>` at line 285 becomes:
  - For image cells: `<img src={mediaUrl}>` — unchanged.
  - For video cells: `<img src={thumbnailUrl ?? undefined}>` where `thumbnailUrl` is read from `thumbnailMap[mediaId]` in the store.
- When `thumbnailUrl` is null/undefined (thumbnail still loading or capture failed): show the existing `<ImageIcon>` placeholder — no spinner needed.
- The `<img>` already has `className="w-full h-full object-contain"` — no visual change needed.

**gridStore.ts** — store addition (D-09, D-10)

- Add `thumbnailMap: Record<string, string>` parallel to `mediaTypeMap`.
- In `addMedia` action: when `type === 'video'`, trigger async first-frame capture. On completion, call `setState` to write the data URL into `thumbnailMap[mediaId]`.
- In `removeMedia` and `clearGrid` and `applyTemplate`: delete the corresponding `thumbnailMap` entries (same cleanup pattern as `mediaTypeMap`).
- Thumbnail capture sequence (D-09, D-11):
  1. Create a hidden `<video>` element (not inserted into DOM).
  2. Set `video.src = blobUrl`, `video.muted = true`, `video.playsInline = true`.
  3. Listen for `loadedmetadata` → set `video.currentTime = 0`.
  4. Listen for `seeked` → draw to a temporary off-screen `<canvas>` → call `canvas.toDataURL('image/jpeg', 0.8)` → update `thumbnailMap`.
  5. Clean up: remove event listeners, set `video.src = ''`.
  6. Edge case: if `seeked` never fires within 2 seconds, skip capture silently (no error state exposed to the user).

## Interaction States

### ActionBar Overflow (CELL-01)
- **Normal:** ActionBar floats above the cell, centered at `top-2`. With `overflow-visible` on the cell container, the bar remains fully visible even when the cell is smaller than the bar's width.
- **Clipped previously:** Bar was hidden by `overflow-hidden` when cell width < bar width. This state is now eliminated.
- **No hover/focus change:** The transition-opacity show/hide behavior on hover is unchanged.

### ActionBar Sizing (CELL-02)
- **Small screen (1366px wide):** `canvasScale` ≈ 0.24 at typical canvas fill. Bar appears at physical 28–30px button size.
- **Large screen (3840px wide):** `canvasScale` ≈ 0.67 at typical canvas fill. Bar appears at physical 28–36px button size (clamped at max 36px).
- **Transition:** No animated transition on resize. Size is computed synchronously from current `canvasScale`.

### Empty Cell Placeholder (CELL-03)
- **Normal:** Icon centered, label below. Both scale with viewport via `clamp()`.
- **Small cell (height < 80px):** Label hidden (`hidden`). Icon remains visible at its minimum clamped size (20px).
- **Drag-over state:** Existing `ring-2 ring-[#3b82f6] ring-inset` drop highlight is unchanged.

### Video Thumbnail (MEDIA-01)
- **Loading:** Sidebar shows `ImageIcon` placeholder (same as empty/no-media state). No spinner.
- **Loaded:** Sidebar shows JPEG still frame via `<img src={thumbnailUrl}>` with `object-contain`.
- **Capture failure (timeout):** Sidebar continues showing `ImageIcon` placeholder. No error message.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps inside buttons |
| sm | 8px | Gap between tray buttons (`gap-2`) |
| md | 16px | Horizontal tray padding (`px-4`) |
| lg | 24px | — |
| xl | 32px | — |
| 2xl | 48px | — |
| 3xl | 64px | — |

Exceptions:
- **Touch target minimum**: 44×44px per CELL-03 (`min-w-[44px] min-h-[44px]`). Fixed size constraint, not a spacing token.
- **Tray height**: ~52px — determined by 44px button height + 4px vertical padding top and bottom (`py-1`).
- **Tray bottom offset**: 60px — positioned directly above the 60px tab strip (`bottom-[60px]`).
- **Tray slide animation offset**: 8px — fade + slide-up distance on enter, slide-down on exit (see Animation Contract).

Source: CONTEXT.md D-02, D-09; REQUIREMENTS.md CELL-03; Phase 23 UI-SPEC (tab strip height 60px).

## Interaction Contract

### Tray Visibility

The tray is visible when `selectedNodeId !== null`. It is hidden (not just invisible) when no cell is selected.

Visibility is controlled by the animation/mount state — the component unmounts or translates off-screen when `selectedNodeId` becomes `null`. See Animation Contract below.

### Layout Stack (bottom of viewport, mobile only)

```
[ canvas area ]
[ MobileCellTray — ~52px, bottom: 60px ]
[ tab strip — 60px, bottom: 0 ]
```

The tray bottom offset is always `60px` regardless of sheet snap state (`collapsed` or `full`). The sheet slides up behind the canvas area; the tray always stays above the tab strip.

Source: CONTEXT.md D-02, D-03, D-04.

### Z-Index Layering

| Layer | z-index | Class |
|-------|---------|-------|
| Canvas | z-0 | (default) |
| MobileSheet | z-40 | `z-40` (existing) |
| MobileCellTray | z-45 | `z-[45]` |
| Overlays / modals | z-50+ | existing |

The tray must render above the sheet (`z-40`) but below any modal/toast overlays. Use `z-[45]` (Tailwind arbitrary value).

Source: CONTEXT.md D-04 (Claude's Discretion for exact z-index).

### Fit Toggle State

| Current `fit` value | Icon shown | `aria-label` |
|---------------------|-----------|--------------|
| `'cover'` | `Minimize2` | `"Switch to contain"` |
| `'contain'` | `Maximize2` | `"Switch to cover"` |

Source: CONTEXT.md D-10; `ActionBar.tsx` existing implementation (note: ActionBar uses `Maximize2` for `contain`→`cover` which means "expand to cover" — same icon logic as desktop).

### Desktop Suppression

The tray uses `md:hidden` to suppress rendering on viewports ≥768px. This matches the established `MobileSheet` pattern and avoids a JS-based `isMobile` conditional.

Source: CONTEXT.md (Claude's Discretion); CONTEXT.md established pattern note.

## Animation Contract

| Transition | Value | Trigger |
|------------|-------|---------|
| Tray enter | `opacity: 0 → 1`, `translateY(8px) → translateY(0)`, duration 300ms, `cubic-bezier(0.32, 0.72, 0, 1)` | `selectedNodeId` becomes non-null |
| Tray exit | `opacity: 1 → 0`, `translateY(0) → translateY(8px)`, duration 300ms, `cubic-bezier(0.32, 0.72, 0, 1)` | `selectedNodeId` becomes null |
| Tray reduced motion | `opacity` transition only (or instant), no translateY movement | `prefers-reduced-motion: reduce` |
| Button hover/active | `transition-colors duration-150` | hover / active state |

The tray animation easing (`cubic-bezier(0.32, 0.72, 0, 1)`) matches the sheet spring curve exactly per CONTEXT.md D-05.

Implementation approach: CSS `transition` on `opacity` and `transform` on the tray container, driven by a `data-visible` attribute or a CSS class toggled by `selectedNodeId`. The component can remain mounted and use CSS-driven visibility, or use a React transition helper — implementer's choice, but the timing values above are fixed.

**Reduced motion**: apply `motion-reduce:transition-none` on the tray container. The tray may appear/disappear instantly with opacity only (no slide).

Source: CONTEXT.md D-05; UI-INTELLIGENCE Animation rules (ease-out enter, ease-in exit; 150–300ms micro-interactions; respect `prefers-reduced-motion`).

## Accessibility Contract

- All 5 buttons have explicit `aria-label` attributes (icon-only buttons require this per WCAG 2.4.6).
- Each button: `min-w-[44px] min-h-[44px]` — WCAG 2.5.5 target size (Level AA). Source: REQUIREMENTS.md CELL-03; CONTEXT.md D-09.
- Gap between buttons: `gap-2` (8px minimum). Source: CONTEXT.md D-09; UI-INTELLIGENCE touch spacing rule.
- Focus ring: `focus-visible:ring-2 focus-visible:ring-[var(--ring)] focus-visible:outline-none` on each button — visible on keyboard/AX navigation, suppressed on pointer. Do NOT use bare `outline-none`.
- `touch-action: manipulation` is already applied globally via `src/index.css` button rule — no per-element override needed.
- Tray container: `role` is implicit (no ARIA role needed on the `<div>` wrapper — buttons inside carry their own semantics).
- Tray container: add `data-testid="mobile-cell-tray"` for Playwright test selectors.
- Reduced motion: `motion-reduce:transition-none` on the tray container.

Source: UI-INTELLIGENCE HIGH severity rules (focus states, touch targets, reduced motion); CONTEXT.md D-09.

# BLOCKING CONSTRAINTS — Read Before Anything Else

> Each constraint below was discovered through failure in Phase 25 (current
> DnD implementation). Acknowledge each one explicitly before writing any new
> DnD code.

- [ ] CONSTRAINT: **Never run TouchSensor + MouseSensor simultaneously.** dnd-kit docs explicitly forbid this; doing so is the primary hypothesized cause of Phase 25's flaky swap and broken 5-zone detection. v1.5 MUST use a single PointerSensor.
- [ ] CONSTRAINT: **Never spread listener props before explicit pointer handlers on the same element.** On LeafNode, `{...dragListeners}` placed before `onPointerDown={handlePan}` let React overwrite dnd-kit's handler. Order matters; put spreads LAST or merge via custom wrapper.
- [ ] CONSTRAINT: **Drag state must not live in gridStore.** Ephemeral drag data in the main store would pollute undo/redo history. Use a separate vanilla Zustand `dragStore`.
- [ ] CONSTRAINT: **Never migrate two DnD engines in parallel.** Swap wholesale — do not have pragmatic/dnd-kit/custom running side-by-side even briefly (Pitfall 10).
- [ ] CONSTRAINT: **Canvas `<LeafNode>` previews cannot use `cloneNode`.** Canvas content doesn't serialize; use `canvas.toDataURL()` → `<img>` for the drag ghost (Pitfall 6 / FEATURES B2).

**Do not proceed with Phase 27 execution until all five boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Dual sensors | TouchSensor + MouseSensor both registered on DndContext | blocking | Single PointerSensor only; lint rule or inline comment on DndContext provider |
| Listener spread before explicit handler | `{...listeners} onPointerDown={x}` on same JSX element | blocking | Code review gate: spreads must come LAST; document on LeafNode |
| Drag state in gridStore | moveCell being invoked optimistically during drag-over | blocking | dragStore is a separate vanilla zustand store; commit only on drop |
| 500ms hold delay | Collides with iOS 17.2+ image-action menu | blocking | Use 250ms + 5px tolerance |
| Parallel DnD engines during migration | Two engines mounted simultaneously | blocking | Phase boundary: remove old `@dnd-kit` wiring in the SAME phase that wires new code |
| `cloneNode` on canvas leaf preview | Ghost renders blank canvas | advisory | Use `toDataURL()` → img in DragPreviewPortal |
| transform: scale() coord math | Viewport↔canvas translations must divide by scale | advisory | Document in computeDropZone.ts; unit test with scale≠1 |

## Animation Contract

This phase introduces three new animated states. All animations must respect `prefers-reduced-motion`.

### 1. Hold-Pulse (pre-activation, 0–500ms)

| Property | Value | Source |
|----------|-------|--------|
| Trigger | React state flag `isHoldPending` set on `touchstart`, cleared on activation or cancel | CONTEXT.md D-05 |
| Transform | `scale(1.03)` pulsing | CONTEXT.md D-05 |
| Duration | 200ms per pulse cycle | UI-INTELLIGENCE: 150–300ms micro-interactions |
| Easing | `ease-in-out` | Symmetric pulse feels natural |
| Implementation | CSS `@keyframes holdPulse` triggered by `data-hold-pending` attribute or class | CONTEXT.md specifics |
| Reduced motion | Skip scale pulse; show static white inset ring only | UI-INTELLIGENCE Result 4 (prefers-reduced-motion) |

```css
@keyframes holdPulse {
  0%, 100% { transform: scale(1); box-shadow: inset 0 0 0 2px rgba(255,255,255,0.0); }
  50%       { transform: scale(1.03); box-shadow: inset 0 0 0 2px rgba(255,255,255,0.35); }
}
@media (prefers-reduced-motion: reduce) {
  .hold-pending { box-shadow: inset 0 0 0 2px rgba(255,255,255,0.35); animation: none; }
}
```

### 2. Lift State (drag active, after 500ms activation)

| Property | Value | Source |
|----------|-------|--------|
| Trigger | `isDragging === true` from `useDraggable` | CONTEXT.md D-06 |
| Transform | `scale(1.08)` | CONTEXT.md D-06 (locked) |
| Opacity | `0.6` | CONTEXT.md D-06 (locked) |
| z-index | `50` | Ensures dragged cell paints above all siblings |
| Transition | `transform 150ms ease-out, opacity 150ms ease-out` | UI-INTELLIGENCE: ease-out for entering states |
| Reduced motion | `opacity: 0.6` only; omit scale transform | UI-INTELLIGENCE Result 4 |

### 3. Zone Overlay Appearance

| Property | Value | Source |
|----------|-------|--------|
| Trigger | `useDndMonitor` detects active drag; existing `activeZone` state drives overlay JSX | CONTEXT.md D-12, existing LeafNode code |
| Enter transition | `opacity 0 → 1` over `100ms ease-out` | Fast; zone overlays should feel instant |
| Exit transition | `opacity 1 → 0` over `80ms ease-in` | Faster exit than enter |
| Reduced motion | No transition; instant appear/disappear | UI-INTELLIGENCE Result 4 |

### 4. Cancel / Snap-Back

| Property | Value | Source |
|----------|-------|--------|
| Trigger | Drag released on non-droppable target | CONTEXT.md D-10 |
| Behavior | @dnd-kit default — cell snaps back to original position | No custom animation required |
| Duration | @dnd-kit default (~250ms ease-out spring) | Default is acceptable |

## Interaction States

### Draggable Cell (LeafNode)

| State | Visual |
|-------|--------|
| Default | No change from existing |
| Hold pending (0–500ms touch) | `holdPulse` animation: scale 1.0→1.03→1.0 + white inset ring; `data-hold-pending` attribute set; `touch-action: none` applied to prevent scroll stealing |
| Drag active (after 500ms) | `scale(1.08)`, `opacity: 0.6`, `z-index: 50`; `touch-action: none` maintained |
| Drag cancelled / snap-back | Styles cleared immediately; @dnd-kit handles positional reset |

### Droppable Cell (non-dragged LeafNode during drag)

| State | Visual |
|-------|--------|
| No drag in progress | No change |
| Drag active, not hovered | No overlay shown (default behavior; zones appear on hover) |
| Hovered — top zone | 4px `#3b82f6` bar at top edge (existing overlay) |
| Hovered — bottom zone | 4px `#3b82f6` bar at bottom edge |
| Hovered — left zone | 4px `#3b82f6` bar at left edge |
| Hovered — right zone | 4px `#3b82f6` bar at right edge |
| Hovered — center zone | `rgba(0,0,0,0.4)` overlay + `ArrowLeftRight` Lucide icon (existing overlay) |

**Zone threshold:** `max(20px, min(cellWidth, cellHeight) * 0.2)` — inherited from existing hit detection at LeafNode.tsx lines 436–442. No change.

## `touch-action` Contract

This is the most critical implementation detail for this phase (source: RESEARCH.md critical section).

| Condition | `touch-action` value | Rationale |
|-----------|----------------------|-----------|
| Global default (Phase 22) | `manipulation` on `button`, `[role="button"]`, inputs, links | Eliminates 300ms tap delay |
| Leaf cell div — default | Not set (inherits browser default `auto`) | Allows scroll and pinch-to-zoom |
| Leaf cell div — hold pending | `none` | Prevents browser scroll from consuming the touch during the 500ms hold window |
| Leaf cell div — drag active | `none` | Maintained for the full drag duration |
| Leaf cell div — drag ended | Cleared back to `auto` / not set | Restores scroll and pinch-to-zoom |

Implementation: apply `style={{ touchAction: (isHoldPending || isDragging) ? 'none' : undefined }}` on the leaf div.

## Copywriting Contract

This phase has no user-facing modal or persistent UI copy. One ephemeral hint label is new.

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable — drag-and-drop is a gesture, not a button |
| Long-press hint (first use only) | "Hold a cell to drag it" |
| Empty state | Not applicable — drag requires cells to exist; empty canvas handled by prior phases |
| Error state | Not applicable — invalid drop silently cancels (cell snaps back per D-10); no error shown |
| Destructive confirmation | Not applicable — no destructive actions in this phase |

**Long-press hint:** Shown as a transient toast or inline tooltip on first drag attempt only (controlled by a one-time `localStorage` flag). Copy: "Hold a cell to drag it". Dismissed automatically after 3 seconds or on first successful drag. Implementation at executor's discretion — the copy is locked.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Keyboard drag | @dnd-kit provides keyboard activation via `KeyboardSensor` — add as a third sensor with default config to support keyboard users (not required by DRAG-01 but required by accessibility standards per UI-INTELLIGENCE Result 1) |
| ARIA during drag | `aria-grabbed` (deprecated) is not used; @dnd-kit sets `aria-pressed` and `aria-roledescription` automatically via `useDraggable` attributes |
| Focus ring | Visible focus ring on cell when keyboard-focused (`focus:ring-2 focus:ring-primary`) must not be suppressed |
| Reduced motion | All animations gated on `prefers-reduced-motion: no-preference` (see Animation Contract above) |
| Touch target size | Draggable cells are full-size grid cells — always ≥44px in the smallest dimension at normal zoom |

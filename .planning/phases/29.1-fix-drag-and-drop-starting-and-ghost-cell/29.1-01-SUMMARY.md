# Phase 29.1-01: Fix drag-and-drop ghost dimensions and accidental activation

**Ghost <img> now renders at uniform-scale ghostW×ghostH matching modifier math; distance-based mouseSensor removed so only 500ms hold activates drag**

## Performance

- **Duration:** ~10 min
- **Started:** 2026-04-19T16:10:00Z
- **Completed:** 2026-04-19T16:20:00Z
- **Tasks:** 2
- **Files modified:** 2

## Accomplishments
- Fixed ghost vertical/horizontal shift: `DragPreviewPortal` now computes `ghostW`/`ghostH` with `Math.min(CAP/sourceW, CAP/sourceH, 1)` — identical to `grabOffsetModifier` — and applies them as inline style. Removed Tailwind `max-w-[var(--ghost-cap)] max-h-[var(--ghost-cap)]` classes that were capping each axis independently, causing axis distortion for non-square cells.
- Fixed accidental drag on quick mouse movement: removed `mouseSensor` (distance:8) from `CanvasWrapper`. Only `touchSensor` (delay:500, tolerance:8) and `keyboardSensor` remain. A quick 8px flick no longer activates drag.
- D-05/D-06 audit: `handleDragMove` pointer derivation (`activatorEvent.clientX/Y + delta.x/Y`) confirmed correct — no code change required.

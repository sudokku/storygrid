# Phase quick Plan 260410-61j: Fix overlay z-index so cell hover never hides overlays — Summary

## What Was Built

Fixed a UX regression where sticker/text/emoji overlays (Phase 13 feature) would disappear behind cell hover highlights and drop-zone indicators. The root cause was `OverlayLayer` rendering at `zIndex: 10` while Tailwind's `z-20` utility class on hovered/active cells was higher. Also added a toolbar toggle so users can hide all overlays during media placement and restore them with one click.

## Changes Made

**src/store/editorStore.ts**
- Added `showOverlays: boolean` to `EditorState` type (after `selectedOverlayId`)
- Added `toggleOverlayVisibility: () => void` to `EditorState` type
- Initialized `showOverlays: true` in store
- Added `toggleOverlayVisibility` action: `set((s) => ({ showOverlays: !s.showOverlays }))`

**src/Grid/OverlayLayer.tsx**
- Subscribed to `showOverlays` via `useEditorStore`
- Added early return `if (!showOverlays) return null` after all hooks, before `sorted` computation
- Raised container `zIndex` from `10` to `30` (above `z-20` cell hover/drop states)

**src/Editor/Toolbar.tsx**
- Subscribed to `showOverlays` and `toggleOverlayVisibility` from `useEditorStore`
- Added `Eye`/`EyeOff` toggle button in right toolbar section, adjacent to safe zone toggle
- Button uses `aria-label` and `title` reflecting current state; `aria-pressed` for accessibility

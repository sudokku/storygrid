# UI Design Intelligence — Phase 30

> Generated by ui-ux-pro-max. Authority reference for platform standards (Apple HIG, Material Design).
> These supplement — do not override — user decisions in CONTEXT.md.

## Touch Interactions & Accessibility

- **Haptic Feedback**: Use `navigator.vibrate(10)` for confirmations. Do NOT overuse — only for meaningful activation events (drag-start, drop-commit). ✓ Matches CROSS-06/07.
- **Touch Target Size**: Minimum 44×44px (`min-h-[44px] min-w-[44px]`). Adjacent targets need ≥8px gap (`gap-2`).
- **Touch Friendly**: Increase touch targets on mobile; desktop-sized targets on mobile are a High severity issue.
- **Focus States**: Visible focus rings required (`focus:ring-2 focus:ring-blue-500`). Never `outline-none` without replacement.
- **Keyboard Navigation**: Tab order must match visual order. All functionality keyboard-accessible.

## Animation & Transition

- **Duration Timing**: 150–300ms for micro-interactions. Never >500ms for UI transitions.
- **Easing**: `ease-out` for entering elements, `ease-in` for exiting. Never `linear` for UI transitions.
- **Reduced Motion**: Must check `prefers-reduced-motion`. Haptic vibration is unaffected but CSS transitions must be removed/reduced.
- **Hover vs Tap**: Hover effects don't work on touch. Primary interactions must use `onClick`/touch handlers.

## Sheet & Drawer Dismissal

- **Back Button**: Preserve navigation history predictably. Do NOT break browser back-button behavior.
- **Sheet Collapse**: Auto-collapse during drag is a UX pattern that clears screen real estate — matches CROSS-08.
- **Navigation Active State**: Current state should be visually indicated.

## Layout & Viewport Safety

- **Viewport Units**: Avoid `100vh` on mobile (browser chrome issues). Prefer `dvh` or `min-h-screen`.
- **Safe Areas**: Account for mobile browser chrome (address bar collapse). Relevant to Pitfall 7 (body.overflow: hidden during drag-start).
- **Content Jumping**: Reserve space for async content. Use fixed dimensions to prevent layout shift.

## Typography (Existing System — Do Not Change)

Phase 30 adds no new typography surfaces. Apply existing design system font scale to any new feedback labels or tray states.

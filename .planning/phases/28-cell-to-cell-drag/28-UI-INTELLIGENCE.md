# UI Design Intelligence — Phase 28

> Generated by ui-ux-pro-max. Authority reference for platform standards (Apple HIG, Material Design).
> These supplement — do not override — user decisions in CONTEXT.md.

## UX — Touch Interaction & Accessibility

- **Touch Target Size (HIGH):** Minimum 44×44px touch targets — `min-h-[44px] min-w-[44px]`
- **Touch Spacing (MEDIUM):** Minimum 8px gap between adjacent touch targets — `gap-2`
- **Focus States (HIGH):** Keyboard users need visible focus rings — `focus:ring-2 focus:ring-blue-500`; never `outline-none` without replacement
- **Haptic Feedback (LOW):** Use `navigator.vibrate(10)` for confirmations/important actions only; don't overuse
- **Touch Friendly (HIGH):** Increase touch targets on mobile vs desktop-sized defaults
- **Hover vs Tap (HIGH):** Use click/tap as primary; don't rely on hover-only for important actions

## UX — Animation & Timing

- **Duration Timing (MEDIUM):** 150–300ms for micro-interactions; never >500ms for UI transitions — `transition-all duration-200`
- **Easing Functions (LOW):** ease-out for entering, ease-in for exiting; never linear
- **Reduced Motion (HIGH):** Respect `prefers-reduced-motion` media query — `@media (prefers-reduced-motion: reduce)`
- **Excessive Motion (HIGH):** Animate 1–2 key elements per view maximum; no decorative bouncing

## UX — Drag & Drop Visual Feedback

- **Hover States (MEDIUM):** Change cursor + subtle visual change on interactive elements — `hover:bg-gray-100 cursor-pointer`
- **Cursor:** `cursor-grab` at rest, `cursor-grabbing` during active drag (per DRAG-01/02 decisions)
- **Ghost opacity:** Source cell dims to 40% during drag (GHOST-07)

## UX — Layout & Spacing

- **Viewport Units (MEDIUM):** Use `dvh` or account for mobile browser chrome; avoid `100vh` on mobile — `min-h-dvh`
- **Content Jumping (HIGH):** Reserve space for async content; use `aspect-ratio` or fixed dimensions
- **Horizontal Scroll (HIGH):** Ensure content fits viewport width — `max-w-full overflow-x-hidden`

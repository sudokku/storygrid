# Milestone v1.4: Mobile-First Overhaul & Instagram Fonts

**Status:** ✅ SHIPPED 2026-04-17
**Phases:** 22–26
**Total Plans:** 6

## Overview

Complete mobile UX overhaul addressing all major friction points on touch devices (header, bottom sheet, cell controls, drag-and-drop) plus Instagram-style font support for text overlays. All phases verified against Playwright audit criteria for 390×844 iPhone 14 viewport.

## Phases

### Phase 22: Mobile Header & Touch Polish

**Goal**: Mobile users can access all primary editor actions and the canvas responds correctly to touch without scroll interference
**Depends on**: Phase 21 (all v1.3 phases complete)
**Requirements**: HEADER-01, HEADER-02, SCROLL-01, SCROLL-02
**Plans**: 1 plan

Plans:
- [x] 22-01: Mobile header 5-button toolbar + CSS touch polish

**Details:**
- Mobile header rebuilt from Export-only to 5 icon buttons: Export, Undo, Redo, Templates, Clear
- All buttons at 44×44px with ≥8px spacing; ExportSplitButton gains `isMobile` prop
- `overscroll-behavior: contain` on CanvasArea prevents pull-to-refresh
- `touch-action: manipulation` on all interactive elements eliminates 300ms tap delay
- Mobile Clear button calls `clearGrid()` directly (no `window.confirm`) — deliberate for touch UX

### Phase 23: Bottom Sheet Redesign

**Goal**: The mobile bottom sheet is reliably accessible and fully scrollable in every state, opened and closed via a toggle button
**Depends on**: Phase 22
**Requirements**: SHEET-01, SHEET-02, SHEET-03, SHEET-04
**Plans**: 1 plan

Plans:
- [x] 23-01: MobileSheet drag→toggle refactor + SheetSnapState narrowing + test updates

**Details:**
- Drag-pill gesture removed; chevron toggle button opens/closes sheet
- `SheetSnapState` narrowed from `half | full | collapsed` to `full | collapsed`
- `useEffect` on `selectedNodeId` auto-expands sheet to full on cell select
- `100dvh` fix for iOS address-bar overflow
- `motion-reduce:transition-none` for reduced-motion accessibility compliance
- Tab strip visible when collapsed so user knows sheet exists

**Plans**: 1 plan

Plans:

**Details:**

### Phase 25: Touch Drag-and-Drop

**Goal**: Mobile users can restructure the grid by long-pressing a cell and dragging it to a new position or slot
**Depends on**: Phase 24
**Requirements**: DRAG-01, DRAG-02, DRAG-03, DRAG-04
**Plans**: 2 plans

Plans:
- [x] 25-01: DndContext wiring + LeafNode @dnd-kit migration + ActionBar cleanup + CSS keyframe
- [x] 25-02: Phase25 tests + phase09 zone test migration + manual verification checkpoint

**Details:**
- `CanvasWrapper` hosts `DndContext` with `MouseSensor(delay:500, tolerance:5)` + `TouchSensor(delay:500, tolerance:5)` + `KeyboardSensor`
- `LeafNode` migrated to `useDraggable` + `useDroppable` + `useDndMonitor`; native file-drop preserved via `dataTransfer.types.includes('Files')` guard
- `DragZoneRefContext` shares pointer position tracking between DndContext host and `useDndMonitor` consumers
- `hold-pulse` CSS keyframe animation (box-shadow only) for visual lift feedback
- `cursor: grab` on hover, `cursor: grabbing` during hold/drag
- Phase 09 zone tests migrated from `fireEvent.dragOver` to `@dnd-kit` mocks
- **Known gap**: Touch drag-and-drop needs an architecture overhaul — interaction model was validated but UX needs revisiting in a future milestone

### Phase 26: Instagram-Style Fonts

**Goal**: Text overlays can use 8 Instagram-aesthetic Google Fonts, each identifiable at a glance in the font picker
**Depends on**: Phase 22
**Requirements**: FONT-01, FONT-02, FONT-03
**Plans**: 1 plan

Plans:
- [x] 26-01: FontPickerList with 8 Instagram fonts + MobileSheet overlay editing

**Details:**
- New `FontPickerList` component: 8 fonts — Bebas Neue, Oswald, Dancing Script, Playfair Display, Space Mono, Pacifico, Barlow Condensed, Caveat
- Consolidated `<link rel="stylesheet">` in `index.html` with `display=swap` (all 8 families, one request)
- Each font name rendered in its own typeface for visual preview (`fontFamily` inline style on button)
- `aria-pressed` for accessible selected-state communication
- Responsive min-height via Tailwind breakpoint classes (`min-h-[44px] md:min-h-[36px]`) — context-agnostic, no mobile prop
- `OverlayPanel` (desktop) and `MobileSheet` overlay branch both wired to `FontPickerList`
- Auto-expand sheet on null→non-null overlay selection pattern

## Milestone Summary

**Known Gaps at Close:**
- `DRAG-01..DRAG-04` implemented but touch drag-and-drop UX needs an overhaul in a future milestone — interaction model works but the overall experience requires redesign
- Requirements traceability table was not updated to `Complete` during execution (documentation gap only — all features shipped)

**Key Decisions:**
- ActionBar stays desktop-only (hover-gated) — mobile gets `MobileCellTray` as a separate component; no changes to portal ActionBar
- `SheetSnapState` narrowed from 3 states to 2 (half snap removed) — simplifies all sheet state logic
- Phase 26 fonts depend only on Phase 22 (not the sheet/tray chain) — loaded independently via Google Fonts CDN
- MouseSensor and TouchSensor unified at `delay:500, tolerance:5` — consistent behavior across pointer devices

**Issues Resolved:**
- Phase 25 pointer event collision and scale overflow fixed via quick task `260416-vgk` post-phase-completion
- 300ms tap delay eliminated globally via `touch-action: manipulation` in CSS

**Technical Debt Incurred:**
- Touch drag-and-drop architecture needs revisiting — works but interaction model is not polished enough for target users; candidate for v1.5

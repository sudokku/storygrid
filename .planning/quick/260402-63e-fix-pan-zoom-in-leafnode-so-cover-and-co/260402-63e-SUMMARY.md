# Quick Task 260402-63e: Fix Pan/Zoom in LeafNode for Cover and Contain Modes

**One-liner:** Natural-dimension-aware absolute-positioned img rendering in LeafNode matching the canvas export formula for correct pan/zoom traversal in both cover and contain modes.

## What Was Done

Replaced the broken `<div>` wrapper + `object-cover` CSS approach with a pixel-accurate `<img>` positioning system that matches the Canvas API export formula in `export.ts`.

### Root cause of the bug

The old cover mode rendered:
```jsx
<div style={{ width: `${scale * 100}%`, height: `${scale * 100}%`, left: ..., top: ... }}>
  <img className="w-full h-full object-cover" />
</div>
```
The inner div has the same aspect ratio as the cell, so `object-cover` always shows the exact same center crop regardless of panX/panY. Pan moved the wrapper but the visible crop did not change.

Contain mode had no pan support at all — the img was static with `object-contain`.

### Fix applied

1. **Natural size tracking:** `imgRef` + `onLoad` handler reads `naturalWidth/naturalHeight` into `naturalSize` state. Resets to `null` on `mediaUrl` change.

2. **Cell size tracking:** `useLayoutEffect` + `ResizeObserver` on `divRef` keeps `cellSize` updated.

3. **`imgRenderParams` useMemo:** Computes `baseW/baseH` (cover or contain fit dimensions) using the same formula as `export.ts drawPannedCoverImage`:
   - Cover: `if (imgAspect > cellAspect) { baseH = ch; baseW = ch * imgAspect; } else { baseW = cw; baseH = cw / imgAspect; }`
   - Contain: inverse — `if (imgAspect > cellAspect) { baseW = cw; baseH = cw / imgAspect; } else { baseH = ch; baseW = ch * imgAspect; }`

4. **Absolute img positioning** (matches canvas formula):
   - `scaledW = baseW * scale`, `scaledH = baseH * scale`
   - `left = (cw - scaledW)/2 + (panX/100) * cw`
   - `top = (ch - scaledH)/2 + (panY/100) * ch`

5. **Pan sensitivity fix:** `handlePointerMove` converts pixel deltas to `% of cell dimensions`:
   - `dxPct = (dx / cw) * 100`, `dyPct = (dy / ch) * 100`
   - Dragging one full cell width always moves panX by 100%, regardless of cell pixel size

6. **Cover mode clamping:**
   - `maxPanX = ((baseW * scale - cw) / 2 / cw) * 100`
   - `maxPanY = ((baseH * scale - ch) / 2 / ch) * 100`
   - Clamps newPanX/newPanY so image edges never leave cell boundary

7. **Fallback rendering:** While `naturalSize` is null (first render before onLoad), falls back to CSS `object-cover`/`object-contain` so something visible appears immediately.

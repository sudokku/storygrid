2. Uses the provided imageCache if given, otherwise creates a new one.
3. Calls `renderNode(ctx, root, ...)` — same as current.

Then refactor `renderGridToCanvas` to delegate to `renderGridIntoContext`:
```typescript
export async function renderGridToCanvas(...): Promise<HTMLCanvasElement> {
  const canvas = document.createElement('canvas');
  canvas.width = width;
  canvas.height = height;
  const ctx = canvas.getContext('2d');
  if (!ctx) throw new Error('Canvas 2D context not available');
  await renderGridIntoContext(ctx, root, mediaRegistry, width, height, settings, videoElements);
  return canvas;
}
```

This preserves backward compatibility for image export (which still calls renderGridToCanvas) while enabling video export to render directly into the stable canvas context.

# UI Design Intelligence — Phase 32

> Generated by ui-ux-pro-max. Authority reference for platform standards (Apple HIG, Material Design).
> These supplement — do not override — user decisions in CONTEXT.md.

## Touch Interaction & Accessibility

### Touch Target Size (High Severity)
- **Do:** Minimum 44x44px touch targets — `min-h-[44px] min-w-[44px]`
- **Don't:** Tiny clickable areas (w-6 h-6 buttons)

### Touch Spacing (Medium Severity)
- **Do:** Minimum 8px gap between touch targets — `gap-2 between buttons`
- **Don't:** Tightly packed clickable elements

### Touch Friendly (High Severity)
- **Do:** Increase touch targets on mobile
- **Don't:** Same tiny buttons on mobile as desktop

### Haptic Feedback (Low Severity)
- **Do:** Use for confirmations and important actions — `navigator.vibrate(10)`
- **Don't:** Overuse vibration feedback

### Keyboard Navigation (High Severity)
- **Do:** Tab order matches visual order; use `tabIndex` for custom order
- **Don't:** Keyboard traps or illogical tab order

### Focus States (High Severity)
- **Do:** Use visible focus rings — `focus:ring-2 focus:ring-blue-500`
- **Don't:** Remove focus outline without replacement (`outline-none` alone)

### Skip Links (Medium Severity)
- **Do:** Provide "skip to main content" link on nav-heavy pages

## Animation & Transitions

### Duration Timing (Medium Severity)
- **Do:** Use 150–300ms for micro-interactions — `transition-all duration-200`
- **Don't:** Animations longer than 500ms for UI elements

### Easing Functions (Low Severity)
- **Do:** Use `ease-out` for entering, `ease-in` for exiting
- **Don't:** Use `linear` for UI transitions

### Reduced Motion (High Severity)
- **Do:** Check `prefers-reduced-motion` media query
- **Don't:** Ignore accessibility motion settings

### Hover vs Tap (High Severity)
- **Do:** Use click/tap handlers for primary interactions — `onClick`
- **Don't:** Rely only on `onMouseEnter` for important actions

### Continuous Animation (Medium Severity)
- **Do:** Use infinite animations for loading indicators only
- **Don't:** Decorative `animate-bounce` on icons

## Navigation, Modal & Drawer Dismissal

### Back Button (High Severity — Mobile)
- **Do:** Preserve navigation history properly — `history.pushState()`
- **Don't:** Break browser/app back button behavior — avoid `location.replace()`

### Active State (Medium Severity)
- **Do:** Highlight active nav item — `text-primary border-b-2`
- **Don't:** All links same style

### Sticky Navigation (Medium Severity)
- **Do:** Add `padding-top` to body equal to nav height when nav is fixed
- **Don't:** Let nav overlap first section content

---

## Layout, Spacing & Viewport

### Viewport Units (Medium Severity — Mobile)
- **Do:** Use `dvh` or account for mobile browser chrome — `min-h-dvh`
- **Don't:** Use `100vh` for full-screen mobile layouts (browser chrome issue)

### Viewport Meta (High Severity)
- **Do:** `<meta name='viewport' content='width=device-width, initial-scale=1'>`
- **Don't:** Missing or incorrect viewport meta tag

### Horizontal Scroll (High Severity)
- **Do:** Ensure content fits viewport width — `max-w-full overflow-x-hidden`
- **Don't:** Content wider than viewport on mobile

### Content Jumping (High Severity)
- **Do:** Reserve space for async content — use `aspect-ratio` or fixed height
- **Don't:** Let images/content push layout around on load

## Shadcn Component Patterns

### Composition (Medium Severity)
- **Do:** `asChild` prop for component composition — `<Button asChild><Link href="/">`
- **Don't:** Wrapper divs for composition

### Custom Components (Medium Severity)
- **Do:** Use `cn()` and `cva` for custom components matching shadcn patterns
- **Don't:** Different styling patterns for custom components

### Semantic Accessibility (High Severity)
- **Do:** Rely on shadcn's built-in ARIA attributes
- **Don't:** Override ARIA attributes with `<div role="button">`

### Component Variants (Medium Severity)
- **Do:** Use variant prop — `<Button variant="destructive">`
- **Don't:** Inline conditional classes — `className={isError ? "bg-red-500" : ...}`

### Individual Imports (Medium Severity)
- **Do:** `import { Button } from "@/components/ui/button"`
- **Don't:** Barrel import from index

### Skeleton for Loading (Medium Severity)
- **Do:** `<Skeleton className="h-4 w-[200px]"/>` matching content layout
- **Don't:** Spinner for content loading states

### Form Pattern (High Severity)
- **Do:** React Hook Form + shadcn `FormField` + `Controller`
- **Don't:** Custom form state management with shadcn inputs

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, drag handle padding |
| sm | 8px | Compact element spacing, sheet header icon gaps |
| md | 16px | Default element spacing, sheet content padding |
| lg | 24px | Section padding within sheet content |
| xl | 32px | Layout gaps (desktop CanvasArea padding, unchanged) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (unused in mobile layout) |

Exceptions:

- Divider hit area: 22px tall/wide (not on scale ŌĆö required for 44px+ effective touch target when combined with finger contact area). Visual line remains 4px.
- Mobile toolbar height: 48px (`h-12`) ŌĆö satisfies 44px minimum touch target for toolbar buttons.
- Bottom sheet collapsed state: 60px visible handle area ŌĆö intentionally off-scale to accommodate both drag handle (8px visual bar) + sheet header row (one line of icon buttons at ~40px).
- Sheet header icon buttons: minimum 44px tap target each (`min-w-[44px] h-11`).

## Typography

Two weights only: 400 (regular) and 600 (semibold).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Sheet content labels, control descriptions |
| Label | 12px (text-xs) | 600 (semibold) | 1.4 | Section headers within sheet, icon button labels below icons. Visually differentiated by size + uppercase + tracking-wide ŌĆö weight 600 reinforces hierarchy. |
| Heading | 16px (text-base) | 600 (semibold) | 1.25 | Sheet header row app name ("StoryGrid" wordmark in mobile toolbar), sheet section titles |
| Display | 20px (text-xl) | 600 (semibold) | 1.2 | Welcome card headline only |

Font family: `var(--font-sans)` = Geist Variable (inherited from existing `html { @apply font-sans }` rule ŌĆö no change required).

No new font sizes are introduced. All sizes use existing Tailwind text-* utilities.

## Interaction Contracts

### Bottom Sheet Snap States

| State | translateY | Visible content | User trigger |
|-------|-----------|-----------------|--------------|
| `collapsed` | `translateY(calc(100% - 60px))` | Drag handle + SheetHeader row only (60px) | Initial state; tap handle to advance; drag down past threshold |
| `half` | `translateY(60vh)` | Full SheetContent (40vh scrollable) | Tap handle from collapsed; drag up past threshold |
| `full` | `translateY(0)` | Full screen height | Drag up from half past threshold |

Animation: `transition: transform 0.3s cubic-bezier(0.32, 0.72, 0, 1)` on the sheet container div. This cubic-bezier matches the iOS sheet feel.

Drag threshold: 50px vertical drag before snap decision fires. Drag direction determines next state (up = next higher state, down = next lower state).

State stored in `editorStore.sheetSnapState: 'collapsed' | 'half' | 'full'`.

### Sheet Header Row (always visible in collapsed state)

Content: four icon-only buttons, left to right:
1. **Templates** ŌĆö `LayoutGrid` icon ŌĆö opens templates popover (same as desktop)
2. **Undo** ŌĆö `Undo2` icon ŌĆö calls `undo()` from gridStore
3. **Redo** ŌĆö `Redo2` icon ŌĆö calls `redo()` from gridStore
4. **Clear** ŌĆö `Trash2` icon ŌĆö calls `clearGrid()` (no confirmation on mobile ŌĆö same behavior as desktop Clear button)

Each button: `min-w-[44px] h-11 rounded-md` for tap target. Icon size: 20px (`size-5`). Color: `var(--muted-foreground)` default, `var(--foreground)` on active press.

### Sheet Content ŌĆö Context-Sensitive

- When `editorStore.selectedNodeId` is null: render `CanvasSettingsPanel` (gap, border radius, background controls ŌĆö reuse existing component unchanged)
- When `editorStore.selectedNodeId` is set: render cell controls ŌĆö Split Horizontal, Split Vertical, Toggle Fit (Cover/Contain), Remove Cell, Clear Media buttons stacked vertically with `gap-2`
- When `editorStore.panModeNodeId` is set: render a full-width "Exit Pan Mode" button at top of sheet content; tap exits pan mode (same as Escape on desktop)

### Canvas Sizing on Mobile (D-12, D-13)

`CanvasArea.tsx` padding change: `p-0 pt-2 md:p-8`. This exposes 100vw width to `CanvasWrapper`'s ResizeObserver, which then computes `scaleByW = containerWidth / 1080`. The canvas fills full screen width with no side letterboxing.

Zoom slider hidden on mobile: `hidden md:flex` wrapper on the zoom control in `Toolbar.tsx`. `editorStore.zoom` locked at 1.0 on mobile (setZoom calls no-op'd when isMobile). `canvasScale = autoFitScale * zoom` ŌåÆ zoom=1.0 ŌåÆ scale is purely autoFitScale.

### Divider Touch Target (D-14)

Hit area div class changes:
- Horizontal divider (between stacked rows): `-top-[11px] left-0 right-0 h-[22px]` (was `-top-[4px] h-[8px]`)
- Vertical divider (between side-by-side cols): `-left-[11px] top-0 bottom-0 w-[22px]` (was `-left-[4px] w-[8px]`)

Visual divider line child element: unchanged. Cursor classes: unchanged.

### Pinch-to-Zoom within Cell (D-11)

Applies only when `isPanMode === true` for that leaf. Two-finger touch gesture on the leaf canvas element.

Touch handler requirements:
- `touchstart`: record `pinchStartDist` and `pinchStartScale`; fires only when `touches.length === 2`
- `touchmove`: compute ratio = currentDist / startDist; clamp `panScale` to [1.0, 3.0]; update via `updateCell()`. Must call `e.preventDefault()` ŌĆö register with `{ passive: false }`.
- `touchend`: clear pinch refs

Single-finger drag in pan mode: existing pointer event handlers already work on touch. No changes needed.

### Mobile Onboarding (D-17, D-18)

Condition: `isMobile && !localStorage.getItem('storygrid_onboarding_done')`.

MobileWelcomeCard layout:
- Position: `fixed inset-0 z-50`
- Backdrop: `bg-black/80`
- Card: `absolute inset-x-4 top-1/2 -translate-y-1/2 bg-[var(--card)] rounded-xl p-6`
- App logo/wordmark at top of card: "StoryGrid" in `text-xl font-semibold text-white`
- Body copy (see Copywriting Contract below)
- "Got it" button: full-width, accent fill (`bg-[var(--sidebar-primary)]`)
- On dismiss: `localStorage.setItem('storygrid_onboarding_done', '1')` ŌĆö same key as desktop overlay

### Cell Swap on Mobile (D-16)

Cell swap uses native HTML5 drag events (dragstart/drop). These work on Android Chrome but NOT on iOS Safari. Document this as a known limitation ŌĆö do not add workaround in this phase. Cell swap remains desktop + Android only.

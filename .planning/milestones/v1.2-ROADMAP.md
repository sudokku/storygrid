## Milestone Summary

### Key Accomplishments

1. **Per-cell visual effects layer** — 6 named preset filters (B&W, Sepia, Vivid, Fade, Warm, Cool) + 4 sliders (brightness/contrast/saturation/blur); applied identically in preview canvas, PNG, and MP4 export via `effectsToFilterString()` on the draw path
2. **Per-cell audio toggle with Web Audio mixing** — `audioEnabled` boolean on LeafNode; ActionBar + Sidebar toggle surfaces; full `AudioContext` → `MediaElementAudioSourceNode` → `MediaStreamAudioDestinationNode` graph; AUD-06 zero-audio MP4 skip path when all cells muted
3. **Text/emoji/sticker overlay system** — Global overlay layer with free-position drag, corner-handle resize, rotation handle; `overlayStore` with 8 Zustand+Immer actions; `stickerRegistry` side-channel mirrors `mediaRegistry`; undo integration; export in PNG and MP4
4. **ffmpeg.wasm eliminated — Mediabunny pipeline** — Removed `@ffmpeg/ffmpeg` + `@ffmpeg/util`; no more 25MB WASM bundle, no COOP/COEP headers required; Mediabunny CanvasSource + AudioBufferSource + OfflineAudioContext mixing; VP9/AVC codec selection; non-fatal AAC fallback toast
5. **Decode-then-encode pipeline (Mediabunny VideoSampleSink)** — Replaced frame-by-frame seeking (99.4% of export time) with upfront decode of all video frames via Mediabunny's higher-level API; sequential per-video decode bounds peak GPU memory; `findSampleForTime()` for O(log n) frame lookup during encode loop
6. **Export Metrics Panel** — 17-field `ExportMetrics` interface (timing, throughput, JS heap, active bitmaps/frames, device info); ref-based 250ms polling (no React re-renders per frame); Shift+M toggle + collapse/expand; feature-flagged via `VITE_ENABLE_EXPORT_METRICS` for zero production cost

### Key Decisions

| Decision | Outcome |
|----------|---------|
| Project Persistence (PERS-01..PERS-12) removed from v1.2 | Repurposed Phase 14 slot for Mediabunny migration — persistence deferred to v1.3 |
| Mediabunny over raw WebCodecs VideoDecoder | Mediabunny provides higher-level decode API (VideoSampleSink) — avoids low-level codec configuration, GPU memory management simpler |
| Feature flag for metrics panel | `VITE_ENABLE_EXPORT_METRICS` + Vite tree-shaking = zero production cost; `.env.development` ships the flag |
| Sequential video decode (not parallel) | Bounds peak GPU memory to one video at a time — prevents OOM on multi-cell grids |
| AUD-08 (persist audio state) deferred | Persistence feature not shipped in v1.2; AUD-08 moves to v1.3 with full PERS block |

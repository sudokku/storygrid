## Animation Contract

All values sourced from CONTEXT.md decisions and UI-INTELLIGENCE.md timing standards.

### MobileCellTray Fade on Drag (D-03, CROSS-08)

- **Trigger:** `dragStore.status === 'dragging'`
- **Properties changed simultaneously:** `opacity: 0` AND `pointer-events: none`
- **Restore:** On drag-end or drag-cancel — `opacity` returns to whatever the prior visibility rule dictates (1 if `selectedNodeId !== null`, 0 otherwise); `pointer-events` restores to `auto` or `none` accordingly.
- **Transition:** Reuse the existing tray transition string already in `MobileCellTray.tsx` line 108:
  `transition: 'opacity 0.3s cubic-bezier(0.32, 0.72, 0, 1), transform 0.3s cubic-bezier(0.32, 0.72, 0, 1)'`
- **Rationale:** `pointer-events: none` prevents ghost tap-throughs on invisible tray buttons (D-03).
- **Reduced motion:** The opacity transition must be set to `transition: none` under `prefers-reduced-motion: reduce`. The `opacity: 0` / `pointer-events: none` values still apply instantly — only the animation is removed.

### MobileSheet Collapse on Drag-Start (D-01, CROSS-08)

- **Trigger:** `dragStore.beginCellDrag` is called (drag lifecycle entry point in `useCellDraggable.ts`).
- **Collapse action:** Read `editorStore.sheetSnapState` → save to `dragStore.prevSheetSnapState` → call `editorStore.setSheetSnapState('collapsed')`.
- **Restore action:** In `dragStore.end()`, read `prevSheetSnapState` → call `editorStore.setSheetSnapState(prevSheetSnapState)` → reset `prevSheetSnapState` to `null`.
- **Sheet animation:** Handled by the existing `MobileSheet` transition (`transform 0.3s cubic-bezier(0.32, 0.72, 0, 1)`). No new CSS added.
- **Timing:** 300ms, `cubic-bezier(0.32, 0.72, 0, 1)` (iOS-style sheet spring). Within the 150–300ms UI-INTELLIGENCE micro-interaction window.
- **Reduced motion:** `MobileSheet` already has `motion-reduce:transition-none` class — collapse/restore still happens instantly with no transition.

### Haptic Pulses (CROSS-06, CROSS-07)

- **Drag-start pulse:** `navigator.vibrate?.(10)` — 10ms single pulse on successful drag activation.
  - Called in `EditorShell.tsx` `onDragStart` callback on `DndContext`.
  - Guard: `?.` optional chaining — no error on Safari/iOS (vibration API unsupported).
- **Drop-commit pulse:** `navigator.vibrate?.(15)` — 15ms single pulse on successful drop commit (after `moveCell` is invoked in `onDragEnd`).
  - Called in `EditorShell.tsx` `onDragEnd` callback, guarded by a truthy `overId` / successful move check.
- **No visual change** — haptics are sensory feedback only.

### iOS Suppression Side-Effects (CROSS-02, CROSS-03, CROSS-04, CROSS-05)

These are invisible by design. No visual or animation changes:

- **CROSS-02** — `touch-action: none` in `useCellDraggable.ts` style return object. Scope: leaf element only. Ancestors retain `touch-action` defaults to preserve sheet scrolling + Phase 5.1 pinch-to-zoom.
- **CROSS-03** — `WebkitTouchCallout: 'none'` in `useCellDraggable.ts` style return object, same element.
- **CROSS-04** — `document.body.style.userSelect = 'none'` / `''` via `useEffect` in `EditorShell.tsx` keyed on `dragStore.status`.
- **CROSS-05** — `document.addEventListener('contextmenu', handler, true)` (capture phase) on drag-start; remove on drag-end.

## Interaction States

### useCellDraggable — draggable LeafNode element

Returned `style` object must contain these additional properties:

```ts
{
  touchAction: 'none',         // CROSS-02 — prevents iOS/Android scroll hijack
  WebkitTouchCallout: 'none',  // CROSS-03 — suppresses iOS long-press image menu
}
```

These are applied only to the draggable `LeafNode` element via the hook's returned style. They MUST NOT be applied to any ancestor element (Pitfall 6 — would break bottom-sheet scrolling and pinch-to-zoom).

### EditorShell — DndContext host side-effects

On `onDragStart`:
1. `document.body.style.userSelect = 'none'` — prevents iOS loupe / text selection (CROSS-04)
2. `document.addEventListener('contextmenu', preventDefault, true)` — capture-phase suppression (CROSS-05)
3. `navigator.vibrate?.(10)` — drag activation pulse (CROSS-06)

On `onDragEnd` and `onDragCancel`:
1. `document.body.style.userSelect = ''` — restore (CROSS-04)
2. `document.removeEventListener('contextmenu', preventDefault, true)` — remove handler (CROSS-05)
3. On `onDragEnd` with successful move only: `navigator.vibrate?.(15)` — drop commit pulse (CROSS-07)

### MobileCellTray — drag-active state

Current `style` object:
```ts
{
  opacity: isVisible ? 1 : 0,
  transform: isVisible ? 'translateY(0)' : 'translateY(8px)',
  pointerEvents: isVisible ? 'auto' : 'none',
  transition: 'opacity 0.3s cubic-bezier(0.32, 0.72, 0, 1), transform 0.3s ...',
}
```

Phase 30 addition — compose `isDragging` from `useDragStore(s => s.status === 'dragging')`:
```ts
{
  opacity: isDragging ? 0 : (isVisible ? 1 : 0),
  pointerEvents: (isDragging || !isVisible) ? 'none' : 'auto',
  // transform and transition unchanged
}
```

Both `opacity` and `pointer-events` change together on a single `isDragging` boolean. No separate state variables.

### dragStore — new field and mutation contract

New field: `prevSheetSnapState: 'collapsed' | 'full' | null` (initial: `null`).

`beginCellDrag` mutation (additions):
1. Read `useEditorStore.getState().sheetSnapState` → store in `prevSheetSnapState`
2. Call `useEditorStore.getState().setSheetSnapState('collapsed')`
3. Set `prevSheetSnapState` in store state

`end()` mutation (additions):
1. Read `prevSheetSnapState` from state
2. If non-null: call `useEditorStore.getState().setSheetSnapState(prevSheetSnapState)`
3. Reset `prevSheetSnapState` to `null` along with the rest of `INITIAL_STATE`

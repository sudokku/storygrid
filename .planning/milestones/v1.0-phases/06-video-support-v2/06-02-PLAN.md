  <action>
Add video rendering support to LeafNodeComponent alongside the existing image rendering. The hidden video element pattern parallels the existing imgElRef pattern.

**1. File input accept attribute:**
Find the `<input>` element with `accept="image/*"` and change to `accept="image/*,video/*"` (per VIDE-01).

**2. Add videoElRef and rafIdRef:**
```typescript
const videoElRef = useRef<HTMLVideoElement | null>(null);
const rafIdRef = useRef<number>(0);
```

**3. Subscribe to isPlaying from editorStore:**
```typescript
const isPlaying = useEditorStore(s => s.isPlaying);
```
Also get `setTotalDuration` for reporting video duration on load.

**4. Detect if current media is a video:**
Read `mediaTypeMap` from gridStore:
```typescript
const mediaType = useGridStore(state => {
  const n = findNode(state.root, id) as LeafNode | null;
  return n?.mediaId ? state.mediaTypeMap[n.mediaId] ?? 'image' : 'image';
});
const isVideo = mediaType === 'video';
```

**5. Video element creation effect (per D-05, D-06, D-07):**
Add a useEffect keyed on `[mediaUrl, isVideo]`. When `isVideo && mediaUrl`:
- Create `document.createElement('video')` with `muted = true`, `playsInline = true`, `loop = true`.
- Set `video.src = mediaUrl` then call `video.load()`.
- On `loadedmetadata`: report duration to editorStore via a helper that computes max across all video durations. Then seek to 0 to draw first frame.
- On `seeked` (once, after the initial seek to 0): call `drawRef.current()` to draw the first frame.
- Store in `videoElRef.current = video`.
- Register in videoRegistry: `registerVideo(id, video, drawRef.current)`.
- Cleanup: set `video.src = ''`, `videoElRef.current = null`, `unregisterVideo(id)`.

When NOT isVideo: ensure `videoElRef.current = null` and `unregisterVideo(id)`.

**6. Update drawRef to handle video:**
The existing `drawRef.current` function draws the image. Extend it to also handle video:
- If `isVideo && videoElRef.current`: use `drawLeafToCanvas(ctx, videoElRef.current, rect, leaf)` — the draw function was updated in Plan 01 to accept HTMLVideoElement.
- If not video and `imgElRef.current`: use existing `drawLeafToCanvas(ctx, imgElRef.current, rect, leaf)` path.

**7. rAF loop effect (per D-06 — runs only while playing):**
Add a useEffect keyed on `[isPlaying, isVideo]`:
```typescript
useEffect(() => {
  if (!isPlaying || !isVideo) {
    // If we just stopped, draw one final still frame
    if (isVideo && videoElRef.current) drawRef.current();
    return;
  }
  function tick() {
    drawRef.current();
    rafIdRef.current = requestAnimationFrame(tick);
  }
  rafIdRef.current = requestAnimationFrame(tick);
  return () => {
    if (rafIdRef.current) {
      cancelAnimationFrame(rafIdRef.current);
      rafIdRef.current = 0;
    }
  };
}, [isPlaying, isVideo]);
```

**8. Video duration reporting:**
When a video loads its metadata, compute the new total duration across all registered videos:
```typescript
import { videoElementRegistry } from '../lib/videoRegistry';

function recomputeTotalDuration() {
  let maxDur = 0;
  for (const video of videoElementRegistry.values()) {
    if (video.duration && isFinite(video.duration)) {
      maxDur = Math.max(maxDur, video.duration);
    }
  }
  useEditorStore.getState().setTotalDuration(maxDur);
}
```
Call `recomputeTotalDuration()` in the `loadedmetadata` handler and in the cleanup function (when a video is unregistered).

**9. Zustand subscribe for pan/zoom redraws:**
The existing code uses `useGridStore.subscribe` and `useEditorStore.subscribe` for 60fps redraws without setState. Ensure the video path also triggers redraws on pan/zoom changes. The existing subscribe pattern should work since `drawRef.current` is updated to handle both image and video.

**10. Do NOT add a `<video>` element to the JSX/DOM (per D-05).** The video element is created programmatically and held only in a ref.

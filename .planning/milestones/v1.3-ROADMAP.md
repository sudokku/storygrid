## Milestone Summary

### Key Accomplishments

1. **hasAudioTrack data model foundation** — required `hasAudioTrack: boolean` on LeafNode with `true` default in `createLeaf()`; undo/redo via `structuredClone` in `pushSnapshot` — no store changes needed; defensive `?? true` reads for legacy persisted state
2. **Instagram-aesthetic named presets with live CSS-filter chip previews** — Clarendon, Lark, Juno, Reyes, Moon, Inkwell replacing generic B&W/Sepia/Vivid/Fade/Warm/Cool labels; each chip applies `effectsToFilterString()` live to a bundled `sample.jpg`; toggle-off resets all 7 effect fields to neutral
3. **Auto-mute detection + BFS multi-file drop** — `HTMLVideoElement` + `loadedmetadata` + `AudioTrackList`/`mozHasAudio` for audio detection (AudioContext cannot parse video containers); `overflowCount` alternation for reliable H/V splits; non-interactive locked VolumeX icon in ActionBar + Sidebar for no-audio cells; fail-open with 5s timeout
4. **Dark PlaybackTimeline restyle** — `bg-black/80 backdrop-blur-sm` container, white controls, 3px scrubber track, `active:scale-150` thumb animation, `cursor-grab/grabbing`; zero TypeScript changes (Tailwind-class-only per PLAY-03)
5. **Live audio preview via Web Audio API** — `useAudioMix` hook; `AudioContext` created synchronously in click handler (autoplay-safe); gain-gated `MediaElementAudioSourceNode` reuse prevents `InvalidStateError`; mute toggle reactive during playback; unmuted cells with `hasAudioTrack: true` contribute audio; muted or no-audio cells are silent

### Key Decisions

| Decision | Outcome |
|----------|---------|
| HTMLVideoElement for audio detection (not AudioContext) | AudioContext.decodeAudioData cannot parse MP4/WebM containers — HTMLVideoElement + loadedmetadata + AudioTrackList is the correct cross-browser path |
| overflowCount replaces depth % 2 for BFS overflow splits | depth stays constant when splitNode uses Case B — counter reliably alternates H/V regardless of which splitNode case fires |
| Gain-gated node reuse (not recreate per play) | Each HTMLVideoElement connects to at most one MediaElementAudioSourceNode per AudioContext lifetime; rapid play/pause uses gain 0/1 toggle — no InvalidStateError |
| hasAudioTrack ?? false in ActionBar (not ?? true) | Old persisted state without the field should show VolumeX locked icon conservatively — fail-closed for display correctness |
| PRESET-05..PRESET-07 (per-cell preset persistence) deferred | No persistence milestone in v1.3; state is ephemeral per session |

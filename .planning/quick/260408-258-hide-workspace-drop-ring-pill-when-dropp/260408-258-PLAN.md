<objective>
Fix bug: workspace blue ring (`ring-4 ring-[#3b82f6] ring-inset`) and `workspace-drop-pill` element remain visible after the user drops files INTO a leaf cell. They currently only clear when dropping on the bare `<main>` background.

Root cause (already diagnosed by orchestrator): `LeafNode.handleDrop` calls `e.stopPropagation()` (LeafNode.tsx:427), so CanvasArea's bubble-phase `onDrop` never fires for cell drops, and `setIsFileDragOver(false)` / `dragCounter.current = 0` never run.

Fix: Add an `onDropCapture` handler on the `<main>` element. The capture phase fires top-down BEFORE the bubble phase reaches LeafNode, so `stopPropagation` does not affect it. The capture handler only resets the visual state — it does NOT consume files. LeafNode still handles file routing via its own `handleDrop`.

Purpose: Eliminate stale drop overlay after intra-cell drops without touching LeafNode or duplicating file processing.
Output: 1 small handler + 1 prop in CanvasArea.tsx; 1 new regression test.
</objective>

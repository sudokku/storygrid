# Milestone v1.5: Unified Drag-and-Drop UX

**Status:** IN PROGRESS
**Phases:** 27–30
**Total Plans:** TBD

## Overview

Wholesale replacement of the Phase 25 cell drag-and-drop implementation with a unified, single-sensor engine that delivers identical, intuitive behavior on both desktop and touch. The new engine is built on `@dnd-kit/core@^6.3.1` with a single `PointerSensor` — no `TouchSensor + MouseSensor` combination. All 42 requirements span six categories: DND engine constraints, drag-start affordances, ghost/preview rendering, drop-target feedback, cancel/abort semantics, and cross-device unification.

Root cause of Phase 25 failures (per quick-task `260416-vgk` post-mortem): JSX prop-order collision between `{...dragListeners}` spread and explicit `onPointerDown`, compounded by two independent pointer sources (`useDndMonitor` + document `pointermove`) producing frame-stale zone coordinates. v1.5 eliminates both failure modes by design: single event source via dnd-kit callbacks, single `PointerSensor`, and ephemeral drag state isolated in a separate `dragStore` that never enters undo history.

**Blocking anti-patterns** (must not be reintroduced — from PITFALLS.md items 1, 2, 4, 10, 11):
1. JSX attribute order collision — spread `{...listeners}` LAST after all explicit handlers (Pitfall 1)
2. Parallel event sources for zone coordinates — all pointer data comes from dnd-kit callbacks only (Pitfall 2)
3. 500ms hold threshold collides with iOS image-action menu — use 250ms + 5px (Pitfall 4)
4. Two DnD engines running simultaneously during migration — Phase 25 wiring removed in the SAME phase as new wiring (Pitfall 10)
5. Drag state in gridStore pollutes undo history — separate vanilla `dragStore` only (Pitfall 11)

**Depends on**: Phase 26 (all v1.4 phases complete)
**Requirements**: DND-01, DND-02, DND-03, DND-05, DND-06, DROP-06, CANCEL-05, CANCEL-06

- [x] 27-02-PLAN.md — TDD computeDropZone pure function (5-zone threshold, no dead space)
- [x] 27-03-PLAN.md — TDD dragStore vanilla Zustand (no Immer, no history, isolated from gridStore)
- [x] 27-04-PLAN.md — Regression test locking gridStore.moveCell no-op guards (DND-05, CANCEL-06)

**Details:**

- Install `@dnd-kit/modifiers@^9.0.0` — the one new dependency for this milestone; used for scale-compensation modifier to correct pointer→canvas coordinate translation through `transform: scale(finalScale)` (DND-06)
- Scaffold `src/dnd/` directory with the full file structure as specified in ARCHITECTURE.md (SUMMARY.md adapter reconciliation applied): `index.ts`, `adapter/dndkit.ts`, `dragStore.ts`, `computeDropZone.ts`, `computeDropZone.test.ts`, `useCellDraggable.ts`, `useCellDropTarget.ts`, `DragPreviewPortal.tsx`, `DropZoneIndicators.tsx` (DND-03)
- Implement `dragStore.ts` as a vanilla Zustand store (no Immer, no history) with shape: `{ status, kind, sourceId, overId, activeZone }` and actions `beginCellDrag`, `setOver`, `end` — explicitly isolated from `gridStore` so pointer-tick writes never enter undo history (DND-02)
- Verify `gridStore.moveCell` (lines 473-494) early-return no-op guards remain untouched and cover: `fromId === toId`, source node not found, target node not found, target not a leaf — drag abort = zero store calls = zero undo entries (DND-05, CANCEL-06)
- Implement `computeDropZone(rect: DOMRect, pointer: {x, y}) → 'center'|'top'|'bottom'|'left'|'right'` as a pure function using viewport-space math (`getBoundingClientRect()` already accounts for `transform: scale()`; no division by `canvasScale` inside this function) (DROP-06)
- Zones tile the cell with no dead space: threshold = `Math.max(20, Math.min(w, h) * 0.2)` ensures every pointer position inside the cell resolves to exactly one of the five zones — pointer at the exact center resolves to `'center'` (CANCEL-05)

- Write `dragStore.test.ts` validating all state transitions: idle→dragging, setOver updates, end resets all fields
- Document the `DND-01` constraint in `adapter/dndkit.ts` file header: "Single PointerSensor only — do NOT add TouchSensor or MouseSensor. See REQUIREMENTS.md DND-01 and PITFALLS.md Pitfall 1."
- No changes to any existing component files in this phase — foundation is purely additive

**Success Criteria** (what must be TRUE):
1. `computeDropZone` tests pass at canvas scales 0.2, 0.5, and 1.0 with no dead-space gaps between zones
2. `dragStore` state transitions test: idle→dragging→setOver→end round-trip asserts correct field values at each step

### Phase 28: Cell-to-Cell Drag

**Goal**: Desktop and touch users can drag any cell and drop it onto any other cell using a single `PointerSensor` engine — REMOVING ALL Phase 25 `@dnd-kit` wiring in this same phase, with no parallel engines mounted simultaneously
**Depends on**: Phase 27
**Requirements**: DND-04, DRAG-01, DRAG-02, DRAG-03, DRAG-04, DRAG-07, GHOST-01, GHOST-02, GHOST-04, GHOST-05, GHOST-06, GHOST-07, DROP-01, DROP-04, DROP-05, DROP-07, CANCEL-03, CANCEL-04, CROSS-01
**Plans**: 5 plans
Plans:
- [ ] 30-01-PLAN.md — Wave 0 test stubs (useCellDraggable, CanvasWrapper, MobileCellTray)
- [ ] 30-02-PLAN.md — dragStore prevSheetSnapState + useCellDraggable style return + LeafNode integration
- [ ] 30-03-PLAN.md — CanvasWrapper side-effects (CROSS-04/05/06/07)
- [ ] 30-04-PLAN.md — MobileCellTray isDragging + complete all tests + full suite gate
- [ ] 30-05-PLAN.md — UAT checkpoint (real-device verification)

**Details:**

- **Remove Phase 25 wiring in this phase** — no parallel engines: delete `DndContext`/`MouseSensor`/`TouchSensor`/`KeyboardSensor` from `CanvasWrapper.tsx`; remove `useDraggable`/`useDroppable`/`useDndMonitor` imports from `LeafNode.tsx`; remove `DragZoneRefContext` inline context from `CanvasWrapper.tsx`; remove all five inline zone JSX blocks from `LeafNode.tsx` (DND-04, Pitfall 10)
- Wire new `DndContext` in `CanvasWrapper.tsx` (or `EditorShell.tsx`) with a **single `PointerSensor`** configured `activationConstraint: { delay: 250, tolerance: 5 }` for touch and `activationConstraint: { distance: 8 }` for mouse — implemented via two separate `PointerSensor` configurations or a custom constraint resolver that branches on event type (DND-01, DRAG-03, DRAG-04)
- Implement `useCellDraggable(leafId)` hook using `useDraggable` from `@dnd-kit/core`: exposes `attributes`, `listeners`, `isDragging`; on drag-start calls `dragStore.beginCellDrag(leafId)` and captures `canvas.toDataURL()` snapshot for ghost; spread `{...listeners}` LAST on the element (after any explicit handlers) per JSX prop-order rule (DRAG-07, GHOST-01, Pitfall 1)
- Implement `useCellDropTarget(leafId)` hook using `useDroppable` from `@dnd-kit/core`: on `onDragOver` fires `computeDropZone(element.getBoundingClientRect(), pointer)` → `dragStore.setOver(leafId, zone)` using only dnd-kit's provided pointer coordinates (no parallel document listener) (Pitfall 2)
- Implement `DragPreviewPortal.tsx` using `@dnd-kit/core`'s `DragOverlay` component, rendered into viewport space (outside the scaled canvas) to avoid double-scaling; displays `<img src={ghostDataUrl}>` at source-cell dimensions — NOT `cloneNode` (GHOST-01, GHOST-06)
- Ghost image follows pointer with grab-point offset preserved via `@dnd-kit/modifiers` custom modifier that corrects for canvas `transform: scale()` — pointer offset from grab point is maintained in viewport coordinates so the ghost does not jump on drag-start (GHOST-02)
- Ghost renders at source-cell dimensions (no 280×280 cap) — width and height match the cell's `getBoundingClientRect()` dimensions (GHOST-04)
- Ghost contains only the cell's media artwork — `DragOverlay` wraps the `<img src={ghostDataUrl}>` with no ActionBar, handles, or selection outline (GHOST-05)
- Source cell dims to 40% opacity during drag via `useDragStore(s => s.sourceId === id && s.status === 'dragging')` selector → `opacity: 0.4` inline style (GHOST-07)
- Implement `DropZoneIndicators.tsx` rendering 5 icon zones (center/top/right/bottom/left) that tile the full cell rectangle with `pointer-events: none` (must not intercept drop events); icons sourced from `lucide-react` (DROP-01)
- Target cell shows 2px accent-color outline when pointer is over any of its 5 zones: driven by `dragStore.overId === id` selector in `LeafNode` (DROP-04)
- No insertion line drawn on edge drops — icons alone convey intent (DROP-05)
- Ghost stays under pointer during hover — no magnetism or snap toward zone centers (DROP-07)
- `cursor: grab` applied via CSS class on all `LeafNode` elements unconditionally (no focus or selection prerequisite) (DRAG-01)
- `document.body` gains `cursor: grabbing` class while drag is active — driven by `dragStore.status === 'dragging'` via a `useEffect` in the `DndContext` host or via `onDragStart`/`onDragEnd` callbacks (DRAG-02)
- Entire cell body is the drag-activation region — no dedicated handle icon or tap-gating for drag start (DRAG-07)
- Release outside the `GridCanvas` element (toolbar, tray, sidebar, browser chrome) triggers cancel — implemented via dnd-kit's `onDragCancel` and/or by checking `location.current.dropTargets.length === 0` on drop, which calls `dragStore.end()` without calling `moveCell` (CANCEL-03)
- Release on origin cell (source === target) triggers snap-back with no undo entry — `moveCell`'s existing `fromId === toId` early-return guard handles the no-op; `onDragEnd` with no valid target animates snap-back (CANCEL-04)
- Single pointer-event stream drives both desktop and touch — `PointerSensor` subsumes both without branching (CROSS-01)
- Mount `<DragPreviewPortal />` once in `EditorShell.tsx` above the canvas
- `LeafNode` wired to `useCellDraggable` and `useCellDropTarget`; `DropZoneIndicators` rendered conditionally when `overId === id`
- Preserve native HTML5 file-drop handlers on `LeafNode` and `CanvasArea` — discriminate via `dataTransfer.types.includes('Files')` guard; file drop stays on its current Phase 8 implementation
- Add `data-dnd-ignore="true"` to `Divider.tsx` hit area + `OverlayLayer.tsx` root; add `canDrag` guard in `useCellDraggable` checking `event.target.closest('[data-dnd-ignore]')`
- Set `pointer-events: none` on `OverlayLayer` selected overlays when `dragStore.status === 'dragging'` (prevents selected overlay from intercepting cell drag hover)
- Run full test suite; update `LeafNode` tests to use new dnd-kit hook pattern (no Phase 25 mock setup); verify `grep -r 'TouchSensor\|MouseSensor\|DragZoneRefContext\|useDndMonitor' src/` returns zero matches

**Success Criteria** (what must be TRUE):
1. Desktop: click-hold (≥8px movement) on any cell initiates drag; releasing on a different cell swaps (center drop) or inserts (edge drop) identically to Phase 9 semantics
2. Touch: 250ms press-and-hold on any cell initiates drag; long-press under 250ms does not trigger drag; releasing on a different cell commits the move
3. `grep -r 'TouchSensor\|MouseSensor\|DragZoneRefContext' src/` returns zero results — no Phase 25 parallel engine remains
4. File drop onto cell and workspace file drop still work — images and videos can be dragged from OS desktop onto any cell
5. Ghost image follows pointer with no jump; source cell shows at 40% opacity; 5-zone indicator appears on hovered target cell with accent outline

### Phase 29: ESC-Cancel + Visual Polish

**Goal**: Every drag interaction has a clearly communicated escape route (ESC key) and polished visual feedback at drag-start, during hover, and on drop commit
**Depends on**: Phase 28
**Requirements**: DRAG-05, DRAG-06, GHOST-03, DROP-02, DROP-03, DROP-08, CANCEL-01, CANCEL-02
**Plans**: TBD

**Details:**

- Implement ESC-to-cancel: window `keydown` listener subscribes on drag-start and unsubscribes on drag-end; ESC triggers dnd-kit's drag cancel flow via the sensor's `cancel()` method or by programmatically calling the `DragCancel` event — `onDragCancel` in `DndContext` calls `dragStore.end()` (CANCEL-01)
- Implement snap-back animation on cancel: 200ms duration with `cubic-bezier(0.15, 1.0, 0.3, 1.0)` easing, animating the ghost's CSS transform from its current viewport position back to the source cell's origin rect (CANCEL-02); snap-back also fires on release-on-origin (CANCEL-04 scenario) — same animation code path
- Verify ESC during a drag does not steal focus from text overlay edit input when one is active — ESC event handler must check `dragStore.status === 'dragging'` before acting; text overlay's own ESC handler fires first (input-focused ESC does not reach window listener if input calls `e.stopPropagation()` or `e.preventDefault()`)
- Ghost opacity is **80%** (`opacity: 0.8`) on the `DragOverlay` `<img>` element — makes ghost semi-opaque as specified (GHOST-03)
- Active zone styling: icon at 100% white (`text-white`), scale 1.1 transform, soft drop-shadow glow — applied to the single zone whose position matches `dragStore.activeZone` when `dragStore.overId === thisLeafId` (DROP-02)
- Inactive zones styling: icon at 30% white (`text-white/30`), no scale transform, no glow — applied to the remaining four zones on any hovered cell (DROP-03)
- Drop-commit flash: landed cell plays a 700ms accent-color background flash (CSS keyframe: accent → transparent) on successful `gridStore.moveCell` commit — driven by `onDragEnd` callback writing a `flashNodeId` field to `dragStore` (DROP-08)
- Drag-start wobble animation: `scale: 1.00 → 1.05 → 1.02` over 150ms with `cubic-bezier(0.34, 1.56, 0.64, 1.0)` (back-out spring easing); fires on `useCellDraggable`'s `isDragging` transition from false → true; implemented as a CSS keyframe class toggled by `isDragging` state (DRAG-05)
- Ghost lift styling: `transform: scale(1.04)` + `box-shadow: 0 8px 24px rgba(0,0,0,0.4)` on the `DragOverlay` container — visible elevation cue distinct from the wobble animation on the source cell (DRAG-06); note: do NOT apply `transform: scale` to the source cell directly (prevents Pitfall 3 — overflow clipping)
- Write Vitest tests for ESC cancel: simulate drag-start → ESC keydown → assert `dragStore.status === 'idle'` and `moveCell` was not called
- Manual UAT checklist: ESC on macOS Chrome, ESC on Firefox, ESC while text overlay is focused, snap-back animation visible at native 60fps

**Success Criteria** (what must be TRUE):
1. ESC during an active drag cancels the drop and triggers a 200ms snap-back animation returning the ghost to its origin cell position
2. Ghost renders at 80% opacity during all drag interactions
3. Active drop zone shows 100% white icon + scale 1.1 + glow; all other zones on the same cell show 30% white icon with no scale
4. Drag-start wobble animation fires (scale 1.00→1.05→1.02 over 150ms) when drag activates; no scale transform on source cell div itself
5. Successful drop commit triggers a 700ms accent-color flash on the landed cell

### Phase 30: Mobile Handle + Tray Polish

**Goal**: Mobile touch interactions feel native: haptic feedback confirms drag-start and drop, the action tray clears the screen during drag, and iOS-specific interference (scroll hijack, image-action menu, context menu, text selection) is fully suppressed
**Depends on**: Phase 28
**Requirements**: CROSS-02, CROSS-03, CROSS-04, CROSS-05, CROSS-06, CROSS-07, CROSS-08
**Plans**: TBD

**Details:**

- Apply `touch-action: none` to the draggable `LeafNode` element only (not to ancestors) to prevent iOS/Android scroll hijack during drag (CROSS-02); scope carefully — ancestors must retain their `touch-action` values to preserve bottom-sheet scrolling and Phase 5.1 pinch-to-zoom (Pitfall 6)
- Apply `-webkit-touch-callout: none` to draggable `LeafNode` elements to suppress the iOS long-press image-action menu (Save/Copy/Share sheet that appears at ~500–700ms) (CROSS-03); combined with the 250ms activation threshold from Phase 28, this provides belt-and-braces iOS protection
- Add `document.body` gaining `user-select: none` on `dragStore.status === 'dragging'` and restoring on drag-end — prevents iOS loupe activation and text-selection interference during drag; implemented via `useEffect` in `DndContext` host component (CROSS-04)
- `contextmenu` events are `preventDefault`'d while drag is active: attach `document.addEventListener('contextmenu', handler, true)` (capture phase) on drag-start; remove on drag-end — suppresses right-click menu mid-drag on desktop and iOS long-press context menu (CROSS-05)
- `navigator.vibrate(10)` fires on successful drag activation (after the 250ms touch hold registers): called inside `onDragStart` callback in `DndContext`; no-op on browsers that don't support vibration API (CROSS-06)
- `navigator.vibrate(15)` fires on successful drop commit: called inside `onDragEnd` after `moveCell` is invoked; no-op on unsupported browsers (CROSS-07)
- `MobileCellTray` and mobile bottom sheet auto-collapse on drag-start: `MobileCellTray` fades to `opacity: 0` when `dragStore.status === 'dragging'`; bottom sheet collapses to `collapsed` snap state on `dragStore.beginCellDrag` and restores previous snap state on `dragStore.end()` — requires storing previous snap state in `dragStore` or as a local ref in the sheet component (CROSS-08)
- Verify `touch-action: none` scope does NOT break bottom-sheet scrolling, Phase 5.1 pinch-to-zoom, or `overscroll-behavior: contain` from Phase 22 — run checklist on real iOS device
- Test `navigator.vibrate` behavior: confirm no-op (no error) on Safari (vibration API not supported on iOS) and confirm 10ms + 15ms pulses are perceptible on Android Chrome
- Real-device UAT required: 250ms long-press clean on iOS Safari, address-bar collapse during drag does not cancel (lock `body.overflow: hidden` during drag-start if needed per Pitfall 7), image-action menu suppressed

**Success Criteria** (what must be TRUE):
1. On Android Chrome: dragging a cell triggers a 10ms haptic on activation and a 15ms haptic on successful drop
2. The `MobileCellTray` becomes invisible (opacity 0) when a drag is active and reappears after drag-end or drag-cancel
3. The mobile bottom sheet collapses to tab-strip on drag-start and restores its previous state on drag-end
4. On iOS Safari: long-pressing a cell with media does NOT show the image-action menu (Save/Copy/Share); drag activates at 250ms
5. Text selection and iOS loupe do not appear during drag on any mobile browser

# UI Design Intelligence — Phase 26

> Generated by ui-ux-pro-max. Authority reference for platform standards (Apple HIG, Material Design).
> These supplement — do not override — user decisions in CONTEXT.md.

## UX: Touch Interaction & Accessibility

### Touch Target Size (High)
- **Do:** Minimum 44x44px touch targets — `min-h-[44px] min-w-[44px]`
- **Don't:** `w-6 h-6` buttons

### Touch Spacing (Medium)
- **Do:** Minimum 8px gap between touch targets — `gap-2` between buttons
- **Don't:** `gap-0` or `gap-1`

### Focus States (High)
- **Do:** Visible focus rings on interactive elements — `focus:ring-2 focus:ring-blue-500`
- **Don't:** `outline-none` without replacement

### Keyboard Navigation (High)
- **Do:** Tab order matches visual order
- **Don't:** Keyboard traps or illogical tab order

### Touch Friendly (High)
- **Do:** Increase touch targets on mobile
- **Don't:** Same tiny buttons on mobile

## UX: Animation & Transitions

### Duration Timing (Medium)
- **Do:** Use 150–300ms for micro-interactions — `transition-all duration-200`
- **Don't:** `duration-1000`

### Easing Functions (Low)
- **Do:** Use `ease-out` for entering, `ease-in` for exiting
- **Don't:** `linear` for UI transitions

### Reduced Motion (High)
- **Do:** Check `prefers-reduced-motion` media query
- **Don't:** Ignore accessibility motion settings

### Loading States (High)
- **Do:** Use skeleton screens or spinners — `animate-pulse` skeleton
- **Don't:** Leave UI frozen with no feedback

### Hover vs Tap (High)
- **Do:** Use click/tap for primary interactions — `onClick` handler
- **Don't:** Rely only on `onMouseEnter` for important actions

## UX: Font Picker / List Selection

### Font Loading (Medium)
- **Do:** Use `font-display: swap` — prevents FOIT
- **Don't:** Invisible text during font load

### Readable Font Size (High)
- **Do:** Minimum 16px body text on mobile — `text-base` or larger
- **Don't:** `text-xs` for body text

### Font Loading (Medium)
- **Do:** Reserve space with fallback font — `font-display: swap + similar fallback`
- **Don't:** Layout shift when fonts load

## Typography

### Bold Typography Mobile Reference
- Scale: 12px labels, 16px body, 22px sub, 32px section, 40px H2, 56px H1, 72px Hero Statement
- Weights: 400 normal, 600–800 for headings
- Letter spacing: -1.5px heroes, -0.5px subheads
- Line height: 1.2–1.5× font size

## shadcn Component Patterns

### Use cn() and cva for custom components (Medium)
- **Do:** `const Custom = ({ className }) => <div className={cn("base", className)}>`
- **Don't:** Inline style-based composition

### Use component variants (Medium)
- **Do:** `<Button variant="destructive">`
- **Don't:** `<Button className={isError ? "bg-red-500" : "bg-blue-500"}>`

### Use Skeleton for loading (Medium)
- **Do:** `<Skeleton className="h-4 w-[200px]"/>`
- **Don't:** `<Spinner/>` for card loading

### Semantic components have built-in ARIA (High)
- **Do:** Rely on component accessibility
- **Don't:** Override ARIA attributes unnecessarily

### Import components individually (Medium)
- **Do:** `import { Button } from "@/components/ui/button"`
- **Don't:** Barrel imports

# UI Design Intelligence — Phase 23

> Generated by ui-ux-pro-max. Authority reference for platform standards (Apple HIG, Material Design).
> These supplement — do not override — user decisions in CONTEXT.md.

## Touch Interaction & Accessibility

### Touch Target Size — HIGH severity
- **Do:** Minimum 44×44px touch targets (`min-h-[44px] min-w-[44px]`)
- **Don't:** Tiny clickable areas (`w-6 h-6` buttons)

### Touch Spacing — MEDIUM severity
- **Do:** Minimum 8px gap between touch targets (`gap-2` between buttons)
- **Don't:** Tightly packed clickable elements

### Touch Friendly Responsive — HIGH severity
- **Do:** Increase touch targets on mobile
- **Don't:** Same tiny buttons on mobile as desktop

### Focus States — HIGH severity
- **Do:** Visible focus rings on interactive elements (`focus:ring-2 focus:ring-blue-500`)
- **Don't:** Remove focus outline without replacement (`outline-none` without alternative)

### Haptic Feedback — LOW severity
- **Do:** Use for confirmations and important actions (`navigator.vibrate(10)`)
- **Don't:** Overuse vibration feedback

## Animation & Transitions

### Easing Functions — LOW severity
- **Do:** Use `ease-out` for entering, `ease-in` for exiting
- **Don't:** Use `linear` for UI transitions

### Duration Timing — MEDIUM severity
- **Do:** 150–300ms for micro-interactions (`transition-all duration-200`)
- **Don't:** Animations longer than 500ms for UI (`duration-1000`)

### Reduced Motion — HIGH severity
- **Do:** Check `prefers-reduced-motion` media query (`@media (prefers-reduced-motion: reduce)`)
- **Don't:** Ignore accessibility motion settings

### Hover vs Tap — HIGH severity
- **Do:** Use click/tap for primary interactions (`onClick` handler)
- **Don't:** Rely only on hover for important actions (`onMouseEnter` only)

### Continuous Animation — MEDIUM severity
- **Do:** Use infinite animations for loading indicators only (`animate-spin` on loader)
- **Don't:** Use for decorative elements (`animate-bounce` on icons)

## Navigation, Modals & Sheet Dismissal

### Back Button — HIGH severity
- **Do:** Preserve navigation history properly (`history.pushState()`)
- **Don't:** Break browser/app back button behavior (`location.replace()`)

### Keyboard Navigation — HIGH severity
- **Do:** Tab order matches visual order
- **Don't:** Keyboard traps or illogical tab order

### Active State — MEDIUM severity
- **Do:** Highlight active nav item with color/underline (`text-primary border-b-2`)
- **Don't:** All links same style

---

## Layout, Spacing & Safe Areas

### Viewport Units — MEDIUM severity
- **Do:** Use `dvh` or account for mobile browser chrome (`min-h-dvh`)
- **Don't:** Use `100vh` for full-screen mobile layouts (`h-screen` on mobile)

### Content Jumping — HIGH severity
- **Do:** Reserve space for async content (`aspect-ratio` or fixed height)
- **Don't:** Let images/content push layout around

### Horizontal Scroll — HIGH severity
- **Do:** Ensure content fits viewport width (`max-w-full overflow-x-hidden`)
- **Don't:** Content wider than viewport

## shadcn/ui Component Patterns

### Use `asChild` for composition — MEDIUM severity
- **Do:** `<Button asChild><Link href="/">`
- **Don't:** Wrapper divs for composition

### Custom components follow shadcn patterns — MEDIUM severity
- **Do:** Use `cn()` and `cva` for custom components
- **Don't:** Different patterns (inline style objects)

### Import components individually — MEDIUM severity
- **Do:** `import { Button } from "@/components/ui/button"`
- **Don't:** Barrel imports

### Use semantic components — HIGH severity
- **Do:** Rely on component built-in ARIA (`<Button>` has button role)
- **Don't:** Override ARIA attributes or use `<div role="button">`

### Skeleton for loading — MEDIUM severity
- **Do:** `<Skeleton className="h-4 w-[200px]"/>` matching content layout
- **Don't:** Spinner for content loading

### Component variants via cva — MEDIUM severity
- **Do:** `<Button variant="destructive">`
- **Don't:** Inline conditional classes (`className={isError ? "bg-red-500" : ...}`)

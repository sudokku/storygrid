<objective>
Apply four surgical touch-interaction fixes to eliminate browser interference with in-app pinch-to-zoom and divider drag on mobile.

Purpose: Two mobile interaction failures — browser zoom competing with the in-app pinch handler, and divider drag being hijacked by browser scroll — are caused by missing or conditional `touch-action: none` CSS and an undersized divider hit target. These four changes fix both failures without changing any app logic.

Output:
- CanvasArea: `touchAction: 'none'` applied unconditionally (removes `sheetOpen` conditional guard + removes its now-unused declarations)
- index.html: viewport meta gains `maximum-scale=1, user-scalable=no`
- Divider: hit area widened from 22px to 40px, `touch-action: none` added via inline style
- divider.test.tsx: `w-[22px]` assertion updated to `w-[40px]`; new assertion verifies `hitArea.style.touchAction === 'none'`
</objective>

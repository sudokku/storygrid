<objective>
Wire the effects module into the single unified draw path so that preview, PNG export, and MP4 export all apply identical effects. This plan extends `drawLeafToCanvas` in `src/lib/export.ts` with a `ctx.filter` application (wrapped in `save/restore`), adds blur edge-bleed mitigation via cell-rect clipping + expanded draw destination, VERIFIES that both PNG and MP4 export call sites forward `leaf.effects` into `drawLeafToCanvas`, and extends the `LeafNode.tsx` Zustand subscriber so the preview canvas redraws when `leaf.effects` changes.

Purpose: locks the WYSIWYG guarantee by hooking at exactly ONE place AND verifying the two export-side callers actually reach that hook with the effects field intact. Without this, preview would update but PNG/MP4 would silently render unfiltered — EFF-08 would break structurally.
Output: updated drawLeafToCanvas; verified renderNode + videoExport call paths; updated LeafNode subscriber; integration tests asserting ctx.filter is set during direct drawLeafToCanvas calls AND during renderNode-driven calls.
</objective>

Documented as expected behaviour.

## Typography

Panel UI chrome typography (sidebar/sheet controls) — matches existing OverlayPanel conventions:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px (text-xs) | 400 (regular) | 1.5 |
| Body / control value | 12px (text-xs) | 400 (regular) | 1.5 |
| Section heading | 12px (text-xs) | 500 (medium), uppercase, tracking-wider | 1.5 |

Font picker rows use each font family as `fontFamily` inline style — size is 14px (text-sm) on desktop, 16px (text-base) on mobile to meet min-16px-on-mobile guideline from UI-INTELLIGENCE.md.

**Font picker row sizes:**
- Desktop: 14px (text-sm), weight 400, line-height 1.4
- Mobile (bottom sheet): 16px (text-base), weight 400, line-height 1.4

## Component Inventory

### New: FontPickerList

Replaces the `<select>` in OverlayPanel.tsx. An inline scrollable column of 8 buttons.

```
[FontPickerList]
  — 8 rows, one per font
  — Each row: <button> with font name rendered in its own typeface
  — Selected row: bg-[#3b82f6] text-white
  — Unselected row: bg-[#2a2a2a] text-neutral-300 hover:bg-[#333333]
  — Click → immediate font apply (no confirm step)
  — No scroll indicator needed (8 items visible in a compact list)
  — Desktop: min-h-[36px] per row (existing control row height pattern)
  — Mobile (bottom sheet): min-h-[44px] per row (touch target compliance)
```

Font order (locked by D-01, FONT-01):
1. Bebas Neue
2. Oswald
3. Dancing Script
4. Playfair Display
5. Space Mono
6. Pacifico
7. Barlow Condensed
8. Caveat

**Geist disposition:** Remove Geist from the visible font list. Existing overlays storing `fontFamily: 'Geist'` will fall back to browser default sans-serif — this is acceptable since Geist is the UI chrome font and reads fine as a fallback. No 9th entry in the picker.

**Source:** D-01, D-02 (CONTEXT.md); FONT-01, FONT-03 (REQUIREMENTS.md).

## Interaction Contract

### Font picker selection

| Trigger | Response |
|---------|----------|
| User taps/clicks a font row | `updateOverlay(id, { fontFamily })` fires immediately — no debounce |
| Font not yet loaded (swap fallback active) | Row renders in system sans-serif temporarily; no loading spinner shown (font-display: swap handles this transparently) |
| Font loaded | Row re-renders in target typeface automatically (browser handles via font-display: swap) |

### Mobile overlay editing

| Trigger | Response |
|---------|----------|
| User taps a text overlay on canvas (mobile) | `selectedOverlayId` set in editorStore; bottom sheet auto-expands to full; sheet label changes to "Text Overlay"; OverlayPanel renders with full controls |
| User taps outside overlay / dismisses selection | `selectedOverlayId` set to null; sheet reverts to cell/canvas controls |
| User taps a cell while overlay is selected | Cell selection takes over; overlay deselected |

### Touch target enforcement (mobile font picker)

Each font picker row in the bottom sheet: `min-h-[44px]` with `touch-action: manipulation`. Gap between rows: 0px (rows are a contiguous list; visual separation via border-bottom `border-[#3a3a3a]` between rows).

## Animation Contract

| Interaction | Duration | Easing | Note |
|-------------|----------|--------|------|
| Font row hover/active | 150ms | ease-out | `transition-colors duration-150` — matches existing OverlayPanel button transitions |
| Bottom sheet expand (overlay selected) | 300ms | cubic-bezier(0.32, 0.72, 0, 1) | Existing MobileSheet transform transition — unchanged |
| Font row selected state (bg color change) | 150ms | ease-out | Same as hover |
| Font swap FOUT | browser-native | n/a | font-display: swap; no JS animation |

Reduced motion: existing `motion-reduce:transition-none` on MobileSheet covers the sheet animation. FontPickerList buttons use `transition-colors` which is similarly short; no special reduced-motion override needed for these.

**Source:** UI-INTELLIGENCE.md (150–300ms micro-interactions, ease-out enter). MobileSheet.tsx existing 300ms cubic-bezier.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Section heading (font picker) | `Font` (matches existing label: `<label className="text-xs text-neutral-400">Font</label>`) |
| Bottom sheet label — overlay selected | `Text Overlay` |
| Bottom sheet label — cell selected | `Cell Settings` (existing, unchanged) |
| Bottom sheet label — no selection | `Canvas Settings` (existing, unchanged) |
| Overlay section heading | `Overlay` (existing, unchanged) |
| Delete overlay button | `Delete overlay` (existing aria-label, unchanged) |
| Empty state — font picker | Not applicable (picker always shows all 8 fonts; no empty state) |
| Error state — font load failure | Not applicable (font-display: swap silently falls back; no error UI shown) |
| Destructive action — delete overlay | No confirmation dialog. Single tap on delete button fires `deleteOverlay()` immediately. Undo is available via Ctrl+Z / the Undo button. |

**Source:** Existing OverlayPanel.tsx labels; D-06 (CONTEXT.md); existing MobileSheet.tsx label logic.

**No new destructive confirmation needed:** Delete overlay is a single-step action with undo available. This matches the existing pattern for cell clear and is consistent with the app's undo-first philosophy.

## Accessibility Notes

- Each font picker row is a `<button>` with `aria-label="{FontName} font"` and `aria-pressed={isSelected}`.
- Focus ring: `focus-visible:ring-2 focus-visible:ring-[#3b82f6]` on each row.
- Tab order: font picker rows in visual top-to-bottom order (Bebas Neue → Caveat).
- Color contrast: `#3b82f6` bg with white text passes WCAG AA at 12px+ (ratio ~4.7:1). Unselected `text-neutral-300` on `#2a2a2a` is ~6:1.

**Source:** UI-INTELLIGENCE.md (focus states, keyboard navigation, touch targets).

# Quick 260408-258: Hide Workspace Drop Ring/Pill When Dropping Into Cell ā€” Summary

Adds an `onDropCapture` handler on `<main>` in `CanvasArea` that resets `dragCounter` and `isFileDragOver` before LeafNode's `e.stopPropagation()` swallows the bubble-phase drop event ā€” fixing the stale blue ring + drop pill that lingered after dropping a file into a leaf cell.

## What Was Done

### Task 1: Add onDropCapture to clear drop visuals before LeafNode swallows the event

**RED:** Added a regression test in `phase08-p02-workspace-drop.test.tsx` that:
1. Renders `CanvasArea` and triggers `dragEnter` to show the pill + ring.
2. Appends a synthetic child div to `workspace-main` with a bubble-phase `drop` listener that calls `e.stopPropagation()` (mimicking `LeafNode.handleDrop`).
3. Dispatches `fireEvent.drop` on the child.
4. Asserts the pill is gone, `ring-4` class is gone, and `autoFillCells` was NOT called (LeafNode owns the cell-drop path).

Test failed as expected ā€” pill stayed visible because no capture-phase reset existed.

**GREEN:** Added in `CanvasArea.tsx`:

```tsx
const handleDropCapture = useCallback((e: React.DragEvent<HTMLElement>) => {
  if (!isFileDrag(e)) return;
  dragCounter.current = 0;
  setIsFileDragOver(false);
}, []);
```

Wired via `onDropCapture={handleDropCapture}` on `<main>`. Bubble-phase `handleDrop` is unchanged and still serves the bare-background drop path. `LeafNode.tsx` is byte-for-byte unchanged.

## Decisions Made

1. **Capture-phase handler is visual-only.** It does not call `autoFillCells` or read `e.dataTransfer.files`. This preserves the existing routing contract: LeafNode handles cell drops, CanvasArea bubble-phase handles bare-main drops, and the new capture handler only cleans up the workspace overlay state.
2. **Guard with `isFileDrag(e)`.** Mirrors the existing `handleDragEnter` / `handleDragLeave` guards so cell-swap drags carrying `text/cell-id` (no `Files` type) are completely unaffected.
3. **No changes to LeafNode.tsx.** The plan explicitly required this; LeafNode's `e.stopPropagation()` semantics for cell-drop file routing remain intact.

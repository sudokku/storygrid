## Scope Note

Phase 31 ships **zero new UI chrome**. It fixes two touch-interaction failures on the existing canvas:

1. Browser-level zoom/scroll competing with the in-app pinch-to-zoom handler.
2. Divider hit areas too small and scroll-hijacked on touch.

All design tokens below are inherited from the existing system. No new components, no new colors, no new typographic elements are introduced.

## Interaction Contract

This section is the primary deliverable for Phase 31, replacing the usual "copywriting" section which is not applicable (no new UI text).

### Touch Action Rules

| Element | CSS Property | Value | Rationale |
|---------|-------------|-------|-----------|
| `CanvasArea` `<main>` | `touch-action` | `none` (unconditional) | D-01: removes conditional guard so browser never intercepts pinch-to-zoom regardless of sheet state |
| Divider hit area `<div>` | `touch-action` | `none` | D-05: prevents browser scroll taking over during divider drag after pointerdown |
| Draggable leaf cell | `touch-action` | `none` | Pre-existing from Phase 30 CROSS-02 — unchanged |

### Viewport Meta

| Attribute | Before | After |
|-----------|--------|-------|
| `<meta name="viewport">` | `width=device-width, initial-scale=1` | `width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no` |

Source: D-02 (CONTEXT.md). Rationale: single-screen app, no scroll content where browser zoom provides utility. Apple HIG and UI-INTELLIGENCE.md confirm this is correct for canvas-only apps.

### Divider Hit Area

| Dimension | Before | After |
|-----------|--------|-------|
| Hit area size | `h-[22px]` / `w-[22px]` | `h-[40px]` / `w-[40px]` |
| Offset (negative) | `−top-[11px]` / `−left-[11px]` | `−top-[20px]` / `−left-[20px]` |
| `data-dnd-ignore` | present | unchanged (pre-existing) |

Source: D-04 (CONTEXT.md). ±4px tolerance permitted if layout demands (user decision).

### DnD Adapter Long-Press Guard

The `src/dnd/adapter/` `pointerdown` handler must abort the 250ms long-press timer if the event target (or any ancestor up to the DnD root) carries `data-dnd-ignore="true"`. This prevents drag activation from firing on dividers.

- Implementation approach: traverse `event.target` → ancestors via `.closest('[data-dnd-ignore="true"]')` before starting the timer.
- If the check already exists in the adapter: verify it covers ancestor traversal, not just direct target.
- Source: D-06 (CONTEXT.md).

### Animation Contract

All timings are inherited from the existing system (Phase 27–30):

| Animation | Duration | Easing | Trigger |
|-----------|----------|--------|---------|
| Drag-start wobble | 150ms | `cubic-bezier(0.34, 1.56, 0.64, 1.0)` | Drag activation |
| Cancel snap-back | 200ms | `cubic-bezier(0.15, 1.0, 0.3, 1.0)` | ESC / pointer-outside / same-cell release |
| Drop-flash | 700ms | `ease-out` | Successful drop commit |
| Micro-interactions (hover, focus) | 150–200ms | `ease-out` entering, `ease-in` exiting | General UI |

No new animations are introduced in Phase 31. Source: `tailwind.config.js` keyframes + REQUIREMENTS.md DRAG-05, CANCEL-02, DROP-08.

Reduced motion: all transitions above must be gated on `prefers-reduced-motion: no-preference` (existing pattern; Phase 31 inherits without change). Source: UI-INTELLIGENCE.md.

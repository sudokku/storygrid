## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DOM → dragStore.setGhost(dataUrl) | `toDataURL()` output crosses from canvas element into in-memory store. Data-URL is base64-encoded pixel data; no code execution vector. |
| dragStore state → DOM (later plans) | `ghostDataUrl` rendered as `<img src>` in DragPreviewPortal — same-origin data-URL, no external fetch. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-01 | I | `ghostDataUrl` field size | accept | `toDataURL()` for a 1080×1920px cell produces a data-URL up to ~5MB. Stored in memory only, cleared on drag-end (`end()` resets to null). Client-side only, no persistence. Low risk. |
| T-28-02 | D | Repeated ghost generation without release | mitigate | `end()` resets `ghostDataUrl` to null via INITIAL_STATE spread. Plan 07 guarantees `end()` is called in `onDragEnd` AND `onDragCancel`. Test in `dragStore.test.ts` asserts `end()` clears the field. |
| T-28-03 | T | Malicious code injects fake ghostDataUrl | accept | Setter is an in-process function; an attacker able to call it already has JS execution. No trust boundary crossed. |
</threat_model>

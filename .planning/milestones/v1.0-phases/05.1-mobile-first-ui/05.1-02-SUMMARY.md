# Phase 05.1 Plan 02: Mobile Cell Controls and Onboarding Summary

Mobile touch controls complete: ActionBar hidden on mobile via CSS, Split H/V buttons in SelectedCellPanel (desktop + mobile sheet), pinch-to-zoom touch handlers in pan mode with passive:false, and MobileWelcomeCard onboarding with shared localStorage key.

## What Was Built

### Task 1: ActionBar Hidden on Mobile, Split H/V Buttons, Pinch-to-Zoom

**ActionBar hidden on mobile** (`src/Grid/LeafNode.tsx`): Added `hidden md:block` to the ActionBar wrapper div className. ActionBar is now CSS-hidden below 768px, fully functional above 768px. Hover logic remains intact for desktop.

**Pinch-to-zoom touch handlers** (`src/Grid/LeafNode.tsx`):
- Added `pinchStartDistRef` and `pinchStartScaleRef` refs to track pinch gesture state
- Registered native `touchstart` (passive:true), `touchmove` (passive:false), `touchend` listeners on the leaf wrapper div when `isPanMode` is true
- `touchmove` calls `e.preventDefault()` to prevent page scroll during pinch
- Uses `Math.hypot()` for two-finger distance calculation
- Scale clamped to [1, 3] via `Math.max(1, Math.min(3, ...))`
- Listeners registered/deregistered via `useEffect` cleanup, same pattern as existing wheel handler (D-11)

**Split H/V buttons in SelectedCellPanel** (`src/Editor/Sidebar.tsx`):
- Added two buttons before the "Actions row" section with `data-testid="split-horizontal"` and `data-testid="split-vertical"`
- Calls existing `split(nodeId, 'horizontal')` and `split(nodeId, 'vertical')` actions
- Visible on both desktop sidebar and mobile sheet (MobileSheet already imports SelectedCellPanel) (D-09)

### Task 2: MobileWelcomeCard and Onboarding Mobile Branch

**MobileWelcomeCard** (`src/Editor/MobileWelcomeCard.tsx`):
- Full-screen overlay with `z-[9999]` and black/80 backdrop
- Headline: "Build your story."
- Body: "Tap a cell to select it. Tap an empty cell to add a photo. Hit Export when you're ready."
- CTA button: "Got it" with `data-testid="welcome-dismiss"` and `bg-[var(--sidebar-primary)]` styling
- `data-testid="mobile-welcome-card"` on outer wrapper (D-17, D-18)

**Onboarding mobile branch** (`src/Editor/Onboarding.tsx`):
- Imports `useMediaQuery` from `../hooks/useMediaQuery`
- Imports `MobileWelcomeCard` from `./MobileWelcomeCard`
- Added `isMobile = useMediaQuery('(max-width: 767px)')` at component top
- Control flow: dismissed → null | isMobile → MobileWelcomeCard | !spotlightRect → null | desktop overlay
- `handleDismiss` (already writes `storygrid_onboarding_done = 'true'`) shared between both paths

**Integration tests** (`src/Editor/__tests__/phase05.1-p02-mobile-controls.test.tsx`):
12 tests across 5 describe blocks:
- Split buttons: render, text content, click calls split with correct args (4 tests)
- MobileWelcomeCard: renders on mobile, headline text, instructions text, dismissal removes card, localStorage set (5 tests)
- Desktop onboarding suppressed on mobile: onboarding-overlay not present on mobile (1 test)
- Already dismissed: neither card nor overlay shown (1 test)
- Shared localStorage key: dismissed via desktop key prevents mobile card (1 test)

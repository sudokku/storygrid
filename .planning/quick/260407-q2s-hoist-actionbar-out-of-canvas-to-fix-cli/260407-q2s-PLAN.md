<objective>
Fix ActionBar clipping by neighbouring cells by hoisting it out of the per-cell DOM subtree into a single React portal mounted at `document.body`. The portal is driven by a new `hoveredNodeId` store field written by `LeafNode` pointer events, and positioned via `getBoundingClientRect()` on a cell element looked up from a new cell registry.

Purpose: Cell `isolate` + sibling paint order + canvas `transform: scale()` make it structurally impossible for an ActionBar living inside a cell to render above a sibling cell. Portal to `document.body` escapes every stacking context and transform containing block in one move (per RESEARCH Option A).

Output:
- New `cellRegistry` module + new `GlobalActionBar` portal component
- `hoveredNodeId` added to editorStore (no selection fallback — hover-only per locked D-01)
- Inline ActionBar block removed from LeafNode; hover handlers route to the store
- ActionBar reverts to fixed `w-8 h-8` sizing (no scale compensation)
</objective>

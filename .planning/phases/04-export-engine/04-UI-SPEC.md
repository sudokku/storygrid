## Component Specifications

### 1. Export Split Button (`ExportSplitButton.tsx`)

Replaces the current placeholder Export button in `Toolbar.tsx` (right group, between safe-zone toggle and Clear button).

**Structure:** Two adjacent `<button>` elements inside a shared `<div>` wrapper with a `1px` vertical divider between them.

- **Left segment** — immediate export using last-used settings:
  - Width: auto (`px-2`)
  - Height: 32px (`h-8`)
  - Contains: `<Download size={16} />` icon + `<span className="text-xs">Export PNG</span>` label (gap 8px / `gap-2`); label reads "Export PNG" when PNG is active format, "Export JPEG" when JPEG is active format
  - States:
    - Default: `text-neutral-300 hover:bg-white/10 transition-colors rounded-l`
    - Disabled (during export): `opacity-40 cursor-not-allowed` — identical pattern to existing Undo/Redo disabled state
  - Tooltip: "Export PNG" (or "Export JPEG" when JPEG is last-used format)

- **Divider between segments:** `w-px h-4 bg-white/20 self-center`

- **Right segment (▼ chevron)** — opens settings popover:
  - Width: 20px (`w-5`)
  - Height: 32px (`h-8`)
  - Contains: `<ChevronDown size={12} />`
  - States:
    - Default: `text-neutral-400 hover:bg-white/10 transition-colors rounded-r`
    - Active (popover open): `bg-white/10`
    - Disabled (during export): `opacity-40 cursor-not-allowed`
  - Tooltip: "Export settings"

- **Shared container:** `flex items-center border border-white/10 rounded` — gives the split button a unified border matching the toolbar's subtle border treatment.

**Keyboard:** Left segment responds to `Ctrl+E` shortcut (wired in EditorShell — Phase 5 shortcut, but the button itself must be keyboard-accessible with standard tab/enter behavior from day one).

### 2. Export Settings Popover

Absolutely-positioned `<div>` toggled by the ▼ button. Closes on outside click via `useEffect` + `mousedown` listener on `document`.

**Position:** `absolute top-full right-0 mt-1` relative to the split button container.

**Dimensions:** `w-56` (224px) width, height by content.

**Visual:**
- Background: `bg-[#1c1c1c]` (matches toolbar)
- Border: `border border-white/10 rounded-md`
- Shadow: `shadow-lg`
- Padding: `p-2` (8px — compact popover)

**Contents (top to bottom):**

1. **Format toggle** — segmented control (PNG / JPEG):
   - Label above: `<span className="text-xs text-neutral-400 mb-2 block">Format</span>`
   - Two buttons side by side: `w-full flex`
   - Each segment: `flex-1 h-7 text-xs rounded transition-colors`
   - Selected: `bg-white/15 text-white`
   - Unselected: `text-neutral-400 hover:text-neutral-300`
   - Container: `flex rounded border border-white/10 overflow-hidden`

2. **Quality slider** (visible only when JPEG selected):
   - Label: `<span className="text-xs text-neutral-400">Quality: {Math.round(quality * 100)}%</span>` — value updates live as slider moves
   - Range input: `min={0.7} max={1.0} step={0.05}` — value stored as 0.0–1.0 float in editorStore
   - Tailwind: `w-full accent-white mt-2`
   - Wrapper: `mt-3` top margin, hidden with `hidden` class when PNG is selected

3. **Download button:**
   - Full-width: `w-full h-8 mt-3 rounded text-xs font-medium bg-white/10 hover:bg-white/15 text-white transition-colors`
   - Label: "Download PNG" when PNG is active format, "Download JPEG" when JPEG is active format
   - Disabled state while exporting: `opacity-40 cursor-not-allowed`

### 3. Toast Notification (`Toast.tsx`)

Minimal custom component. Fixed position, bottom-right. No external library.

**Position:** `fixed bottom-4 right-4 z-50`

**Visual:**
- Container: `flex items-center gap-2 px-3 py-2 rounded-md text-sm font-medium shadow-lg`
- Background: `bg-[#1c1c1c] border border-white/10`
- Min width: `min-w-[200px]` Max width: `max-w-[300px]`
- Transition: `opacity` + `translateY` on mount/unmount — `transition-all duration-200`

**States:**

| State | Icon | Text | Background |
|-------|------|------|------------|
| Preparing | spinner (animated `animate-spin`, `Loader2` icon 14px) | "Preparing…" | `bg-[#1c1c1c]` |
| Exporting | spinner (`Loader2` 14px) | "Exporting…" | `bg-[#1c1c1c]` |
| Success | — | dismissed (unmounts automatically) | — |
| Error | `AlertCircle` 14px, `text-destructive` | "Export failed. Try again." | `bg-[#1c1c1c]` |
| Video blocked | `AlertCircle` 14px, `text-destructive` | "Export not available: remove video cells first." | `bg-[#1c1c1c]` |

- Success state: toast dismisses automatically when download begins (no "success" toast shown — the download itself is the confirmation).
- Error toast: persists until user clicks the "Try again" button or dismisses. "Try again" is an inline text button (`text-xs underline text-destructive ml-auto`) that re-triggers the export.
- Toast unmounts (not hides) after success or dismiss.

**Icon text color:** `text-neutral-400` for spinner, `text-destructive` (var(--destructive)) for error icon only.

### 4. ExportSurface (`ExportSurface.tsx`)

Not visible to the user. Contract governs what it must NOT render in export mode.

**Position and size:**
```css
position: absolute;
left: -9999px;
top: 0;
width: 1080px;
height: 1920px;
visibility: hidden;
overflow: hidden;
```

Applied via inline style (not Tailwind) — these are exact pixel values required by html-to-image; arbitrary Tailwind values are acceptable but inline style is simpler and less error-prone.

**aria-hidden:** `true` — never read by screen readers.

**Export mode suppressions** (controlled via `ExportModeContext` set to `true`):
- LeafNode: no ActionBar, no selection ring (`ring-2 ring-blue-500`), no hover dim overlay, no `onMouseEnter`/`onMouseLeave` handlers
- ContainerNode: no Divider component rendered
- No cursor changes, no pointer events needed

| Condition | Left Segment | Right Segment (▼) | Popover |

### Quality Slider Visibility

| Selected Format | Slider Visible |
|-----------------|---------------|
| PNG | No (`hidden`) |
| JPEG | Yes |

### Toast Lifecycle

```
Export triggered
  → Toast mounts: "Preparing…" (first toPng call begins)
  → Toast updates: "Exporting…" (second toPng call begins)
  → Success: Toast unmounts (download triggers)
  → Error: Toast updates to error state; persists until dismissed or "Try again"
```

Toast is a single persistent element that transitions between states — it does NOT stack multiple toasts. Only one export can be in progress at a time (button disabled during export prevents this).

## Accessibility Contract

- Export split button left segment: `aria-label="Export PNG"` (or "Export JPEG") + keyboard Enter/Space
- Export split button right segment: `aria-label="Export settings"` + `aria-expanded={popoverOpen}`
- Popover: `role="dialog"` + `aria-label="Export settings"` — closes on Escape key via `keydown` handler
- Quality slider: `<input type="range" aria-label="Export quality">` — value announced via aria-valuetext: "90 percent"
- Format toggle: two `<button role="radio">` elements inside a `role="radiogroup"` wrapper
- Toast: `role="status"` for preparing/exporting states; `role="alert"` for error state
- ExportSurface: `aria-hidden="true"` — excluded from all accessibility trees

## Implementation Notes for Executor

1. **ExportSurface placement:** Mount as a sibling of `<CanvasArea>` inside `EditorShell`, NOT inside `CanvasArea` or any element with `transform: scale()`. See RESEARCH.md Pitfall 2.

2. **ExportModeContext:** Create `src/Grid/ExportModeContext.tsx`. Provide `true` inside ExportSurface tree only. Default context value is `false` so existing preview components are unaffected without any code changes until they explicitly consume the context.

3. **editorStore additions:** Add `isExporting: boolean`, `exportFormat: 'png' | 'jpeg'`, `exportQuality: number` (default `0.9`). See RESEARCH.md Pattern 5 for full type additions.

4. **Double-call pattern:** Both `toPng()` and `toJpeg()` must be called twice (first result discarded). This applies to both format paths. See RESEARCH.md Pattern 2.

5. **Popover outside-click:** `useEffect` registers `mousedown` on `document` when popover open; removes on close or unmount. Check click target is not inside popover ref before closing.

6. **Export button disabled in finally:** `setIsExporting(false)` must be in `finally` block, not only in success path. See RESEARCH.md Pitfall 7.

7. **No CSS `display: none` on ExportSurface:** Use `visibility: hidden` only. `display: none` prevents html-to-image from capturing the element.

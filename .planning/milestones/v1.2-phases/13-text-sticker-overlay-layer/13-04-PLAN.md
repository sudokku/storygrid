    STEP 1 — Replace src/Editor/EmojiPickerPopover.tsx stub with full implementation:
    ```tsx
    import { useState, useEffect } from 'react';
    import { useOverlayStore } from '../store/overlayStore';

    interface EmojiPickerPopoverProps {
      onClose: () => void;
    }

    export function EmojiPickerPopover({ onClose }: EmojiPickerPopoverProps) {
      const [PickerComponent, setPickerComponent] = useState<React.ComponentType<any> | null>(null);
      const [emojiData, setEmojiData] = useState<unknown>(null);
      const [error, setError] = useState<string | null>(null);

      useEffect(() => {
        let cancelled = false;
        (async () => {
          try {
            const [pickerMod, dataMod] = await Promise.all([
              import('@emoji-mart/react'),
              import('@emoji-mart/data'),
            ]);
            if (cancelled) return;
            setPickerComponent(() => pickerMod.default ?? pickerMod.Picker);
            setEmojiData(dataMod.default ?? dataMod);
          } catch (e) {
            if (!cancelled) setError('Failed to load emoji picker');
          }
        })();
        return () => { cancelled = true; };
      }, []);

      if (error) return <div className="p-4 text-red-400">{error}</div>;
      if (!PickerComponent || !emojiData) return <div className="p-4 text-zinc-400">Loading…</div>;

      return (
        <PickerComponent
          data={emojiData}
          theme="dark"
          onEmojiSelect={(emoji: { native: string }) => {
            useOverlayStore.getState().addOverlay({
              type: 'emoji',
              x: 540, y: 960,
              width: 128,
              rotation: 0,
              char: emoji.native,
            });
            onClose();
          }}
        />
      );
    }
    ```
    Key points: lazy import happens inside useEffect; loading spinner prevents blank picker (pitfall 7); onEmojiSelect reads `emoji.native` per RESEARCH.md A1 assumption; theme 'dark' matches editor.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User file upload → stickerRegistry | HIGH attack surface — SVG XSS via script tags, event handlers, foreignObject |
| emoji-mart lazy chunk | Third-party code lazy-loaded — trusted but bundled |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-13-09 | XSS via SVG upload | StickerUpload SVG handling | HIGH | mitigate | DOMPurify.sanitize with `USE_PROFILES: { svg: true, svgFilters: true }` applied BEFORE storage in stickerRegistry. Unit tests (Task 1, 5 cases) assert script tags, event handlers, and foreignObject script payloads are stripped. |
| T-13-10 | Prototype pollution via emoji-mart data | EmojiPickerPopover | medium | mitigate | emoji-mart data is a static JSON bundle shipped with @emoji-mart/data; no user-controlled keys enter the picker. We do NOT spread emoji data into any overlay object — only `emoji.native` (a string) is read and stored on EmojiOverlay.char. |
| T-13-11 | DoS via large PNG sticker upload | StickerUpload PNG handling | medium | accept | File size is bounded by browser FileReader; stickerRegistry is side-channel (not in history snapshots) so a large sticker cannot bloat undo history. Accept without explicit size cap in v1.2 — if users report OOM we add a size cap in a follow-up. |
| T-13-12 | Spoofing via file.type header | StickerUpload type detection | medium | mitigate | PNG path uses FileReader.readAsDataURL which produces a data URI containing the actual MIME — browsers validate PNG signature. SVG path reads as text and sanitizes, so even if a malicious file claims image/svg+xml but contains HTML, DOMPurify neutralizes it. |
</threat_model>

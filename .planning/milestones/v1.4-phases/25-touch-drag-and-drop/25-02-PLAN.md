**src/test/phase09-p03-leafnode-zones.test.ts** — Update to work with @dnd-kit:

The existing tests use `fireEvent.dragOver` with native drag events and `dataTransfer.types` including `'text/cell-id'`. After Plan 01, the native drag handlers are removed. The zone overlays are now driven by `useDndMonitor.onDragOver`.

Strategy: Mock `@dnd-kit/core` the same way as in phase25 tests. Instead of `fireEvent.dragOver`, directly invoke `mockDndMonitorCallbacks.onDragOver(...)` to set zones. The assertions on zone overlay testids (`edge-line-top-{id}`, `swap-overlay-{id}`, etc.) remain the same — only the trigger mechanism changes.

Specific changes:
1. Add the same `vi.mock('@dnd-kit/core', ...)` block as in phase25 tests
2. Replace all `fireEvent.dragOver(leafDiv, ...)` calls with direct invocation of `mockDndMonitorCallbacks.onDragOver({ over: { id: LEAF_ID }, active: { id: SOURCE_ID } })`
3. For zone-specific tests, mock `getBoundingClientRect` on the leaf div AND set `pointerPosRef` coordinates. Since `pointerPosRef` is internal to LeafNode, instead mock `document.addEventListener('pointermove', ...)` to capture the handler, then dispatch a pointermove event with specific clientX/clientY before invoking onDragOver.
4. Replace `fireEvent.drop(...)` usage with assertions on the CanvasWrapper's `onDragEnd` callback behavior (tested in phase25 DRAG-04 tests — the zone tests only need to verify overlay rendering, not drop dispatch).
5. Remove `fireEvent.dragLeave(...)` calls — replaced by `mockDndMonitorCallbacks.onDragEnd()` or `onDragCancel()`.
6. Keep all overlay testid assertions unchanged (`edge-line-top-{id}`, `swap-overlay-{id}`, etc.).

<objective>
Fix two UAT-confirmed drag-and-drop bugs in DragPreviewPortal.tsx and CanvasWrapper.tsx.

Bug 1 (ghost vertical shift): The ghost <img> has inline `style={{ width: sourceW+'px', height: sourceH+'px' }}` plus Tailwind `max-w-[var(--ghost-cap)] max-h-[var(--ghost-cap)]` classes that cap each axis at 200px independently. For a 400×300 cell this causes the browser to render a 200×200 ghost while the modifier computed ghostH=150, producing a vertical offset that grows the further down the user grabs. Fix: compute ghostW/ghostH in the component using the same `Math.min(CAP/sourceW, CAP/sourceH, 1)` formula as the modifier and set `style={{ width: ghostW+'px', height: ghostH+'px' }}`. Remove the Tailwind cap classes.

Bug 2 (accidental drag on mouse movement): Two PointerSensor instances are registered — `touchSensor` (delay:500, tolerance:8) and `mouseSensor` (distance:8). dnd-kit evaluates sensors in order and the distance sensor fires immediately on any 8px movement, bypassing the 500ms hold intention. Fix: remove `mouseSensor` declaration and remove it from `useSensors(...)`.

Audit result (D-05/D-06): `handleDragMove` pointer derivation (`activatorEvent.clientX/Y + delta.x/Y`) is correct and independent of ghost position — no code change required.

Purpose: Restore the precise grab-point UX users expect: the ghost appears exactly where they clicked, and drag only activates on a deliberate hold.
Output: Two modified files with no new dependencies.
</objective>

### Human Verification Required

#### 1. Bottom Sheet Drag Gestures on Real Touch Device

**Test:** Open on iPhone or Android phone. Drag the sheet handle upward past 50px. Verify snap to half-state; drag again to full-state; drag down to collapse.
**Expected:** Smooth CSS transition (`cubic-bezier(0.32, 0.72, 0, 1)`); each snap state shows the correct sheet height; no page scroll interference during drag; transition disabled during active drag.
**Why human:** Touch pointer capture and passive event behavior differ between jsdom and mobile browsers. The `setPointerCapture` path requires a real PointerEvent from hardware.

#### 2. Canvas Fills 100vw on Phone (D-12)

**Test:** Open on a real phone at < 768px viewport. Verify the canvas occupies full screen width edge-to-edge with no horizontal scroll, side gap, or letterboxing.
**Expected:** Canvas touches both screen edges; bottom sheet collapsed handle (60px) sits below canvas; no gap on either side.
**Why human:** CSS layout at device pixel ratios and safe areas (iPhone notch) requires visual verification.

#### 3. Double-Tap Enters Pan Mode + Pinch-to-Zoom on Touch (D-11)

**Test:** On a phone, upload an image to a cell. Double-tap the cell. Verify pan mode yellow ring appears and sheet shows Exit Pan Mode button. Drag to reposition image. Pinch to zoom (expect scale 1x–3x).
**Expected:** Pan mode activates; image repositions smoothly; pinch scales image within [1x, 3x]; Exit button exits pan mode.
**Why human:** `dblclick` simulation differs from real touch double-tap; `passive: false` touchmove behavior requires real device.

#### 4. iOS File Picker Camera Option (D-15)

**Test:** On iPhone, tap an empty cell. Verify native iOS photo picker appears with both photo library and camera options.
**Expected:** iOS system sheet appears with "Photo Library" and "Camera" choices.
**Why human:** Requires iOS hardware. The code path (`input[type=file] accept="image/*"`) is correct but camera option display is OS-specific and varies by iOS version.

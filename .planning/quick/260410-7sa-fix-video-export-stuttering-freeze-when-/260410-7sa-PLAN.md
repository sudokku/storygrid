        destroyExportVideoElements(exportVideoElements);

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| export pipeline → MediaRecorder | Canvas pixel data captured by browser's MediaRecorder; no external trust boundary |
| video blob URLs → HTMLVideoElement | Blob URLs from mediaRegistry are same-origin; no cross-origin risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-7sa-01 | Denial of Service | isRendering guard | accept | If a single frame takes extremely long (e.g., OOM), the guard holds the lock and the export stalls rather than crashing. This is acceptable — the alternative (concurrent writes) causes corruption. |
| T-7sa-02 | Denial of Service | captureStream(0) + requestFrame | accept | If requestFrame is never called (early throw before any render), MediaRecorder gets zero frames → empty/invalid Blob. Existing error paths call recorder.stop() which triggers onstop → Blob is returned with zero chunks. Already handled. |
| T-7sa-03 | Tampering | fastSeek | accept | fastSeek targets nearest keyframe, not exact timestamp. Audio/video drift of up to one keyframe interval (~0.5s) is acceptable per quality-vs-gap tradeoff documented in RESEARCH.md. |
</threat_model>

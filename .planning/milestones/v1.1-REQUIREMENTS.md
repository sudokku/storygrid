# Requirements: StoryGrid v1.1 — UI Polish & Bug Fixes

> Milestone goal: Fix visual bugs and polish the editing experience — no new features.
> Phase numbering continues from v1.0 (last phase: 6 → v1.1 starts at Phase 7).

---

## Milestone Requirements

### A — Cell Action Bar

- [x] **CELL-01:** User can always access cell top-bar controls regardless of cell size — Validated in Phase 07, reinforced in Phase 10 via portal architecture (createPortal to document.body escapes per-cell stacking contexts; `isolate` removed from LeafNode root)
- [x] **CELL-02:** Cell action bar controls maintain a stable, consistent size across all screen resolutions — Re-validated in Phase 10 via portal-aware fixed `w-16 h-16` (64px) sizing. The original "vw/vh" wording was superseded by the portal architecture where scale compensation is unnecessary.

### B — Sidebar / Media

- [x] **MEDIA-01:** User sees a first-frame thumbnail for video cells in the sidebar — Validated in Phase 07 (display path); Phase 10 closed the sidebar Replace upload-input gap (`accept="image/*,video/*"` + handleFileChange video branching)

### C — Safe Zone

- [x] **CANVAS-01:** When "Show Safe Zone" is enabled, unsafe areas display a visually distinct overlay — Validated in Phase 08: canvas-workspace-ux

### D — Templates

- [x] **TPL-01:** User can apply a preset template without a confirmation dialog — Validated in Phase 08: canvas-workspace-ux

### E — Drop Zone

- [x] **DROP-01:** User can drop media files anywhere in the workspace area — Validated in Phase 08: canvas-workspace-ux
- [x] **DROP-02:** The workspace drop zone provides clear visual feedback when a file is dragged over it — Validated in Phase 08: canvas-workspace-ux

### F — Empty Cell Empty State

- [x] **CELL-03:** Empty cell placeholder icon and label scale proportionally on large screens using vw/vh-based sizing — Validated in Phase 07: cell-controls-display-polish

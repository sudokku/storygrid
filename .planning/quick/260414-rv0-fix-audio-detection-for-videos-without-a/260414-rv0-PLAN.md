# Quick Task 260414-rv0: Fix Audio Detection for Chrome

## Root Cause

Chrome does not implement `HTMLMediaElement.audioTracks`. Both `audioTracks` and
`mozHasAudio` are `undefined` in Chrome, so `detectAudioTrack()` always hits the
fail-open branch (`resolve(true)`). Every video — including silent ones — is flagged
as having audio, causing the interactive volume toggle to appear instead of the
dimmed VolumeX icon.

## Fix

Insert a third detection path between `mozHasAudio` and the fail-open fallback:

```
captureStream() / mozCaptureStream() → stream.getAudioTracks().length > 0
```

Chrome supports `captureStream()` and `MediaStream.getAudioTracks()` returns the
correct tracks even before playback starts.

Detection order after fix:
1. `audioTracks` API → Safari (already correct)
2. `mozHasAudio` → Firefox (already correct)
3. `captureStream()` / `mozCaptureStream()` → Chrome + Firefox fallback (NEW)
4. Fail-open `resolve(true)` → only for browsers that support none of the above

This is a single-file fix with three changes to LeafNode.tsx:

**1. Add imports and state/refs (top of component, after line 35):**

Add inside the component (before the early return at line 37):
- `imgRef = useRef<HTMLImageElement>(null)` — ref for the rendered img element
- `[naturalSize, setNaturalSize] = useState<{ w: number; h: number } | null>(null)` — image natural dimensions
- `[cellSize, setCellSize] = useState({ w: 0, h: 0 })` — cell container dimensions
- `handleImgLoad` callback that reads `imgRef.current.naturalWidth/naturalHeight` into naturalSize

Add a `useLayoutEffect` (after the existing wheel useEffect) that:
- Reads `divRef.current.clientWidth/clientHeight` into cellSize
- Creates a ResizeObserver on divRef.current to keep cellSize updated
- Returns cleanup disconnecting the observer

Add a `useEffect` that resets `naturalSize` to null when `mediaUrl` changes.

Add `imgRenderParams` useMemo that computes, given naturalSize and cellSize:
- For cover: baseW/baseH = the cover-fit dimensions (image scaled so it fully covers the cell, preserving aspect ratio)
- For contain: baseW/baseH = the contain-fit dimensions (image scaled to fit inside the cell, preserving aspect ratio)
- Returns `{ baseW, baseH, cw: cellSize.w, ch: cellSize.h }` or null if dimensions not yet available

The cover math must match export.ts drawPannedCoverImage (lines 199-209):
```
if (imgAspect > cellAspect) { baseH = ch; baseW = ch * imgAspect; }
else { baseW = cw; baseH = cw / imgAspect; }
```

**2. Fix handlePointerMove (replace lines 132-140):**

Convert pixel deltas to percentage of cell dimensions:
```
dxPct = (dx / cw) * 100
dyPct = (dy / ch) * 100
newPanX = panStartRef.current.panX + dxPct
newPanY = panStartRef.current.panY + dyPct
```

For cover mode, clamp so image edges cannot leave the cell:
```
maxPanX = ((baseW * scale - cw) / 2 / cw) * 100
maxPanY = ((baseH * scale - ch) / 2 / ch) * 100
```
Clamp newPanX to [-maxPanX, maxPanX], newPanY to [-maxPanY, maxPanY].

For contain mode, allow free panning within [-100, 100] range (matches current behavior intent).

Add `imgRenderParams` and `node?.fit` to the useCallback dependency array.

**3. Replace media rendering block (lines 190-214):**

When `imgRenderParams` is available, render a single `<img>` with:
- `ref={imgRef}`, `onLoad={handleImgLoad}`
- `position: absolute`
- `width: scaledW = baseW * scale` px
- `height: scaledH = baseH * scale` px
- `left: (cw - scaledW)/2 + (panX/100) * cw` px
- `top: (ch - scaledH)/2 + (panY/100) * ch` px

This matches export.ts formula: center the natural-aspect cover/contain image, then translate by panX% of cell width.

When `imgRenderParams` is null (natural dimensions not yet loaded), render a fallback `<img>` with the old object-cover/object-contain classes and `ref={imgRef}` + `onLoad={handleImgLoad}` so dimensions get captured on first load.

The empty-media branch (ImageIcon + "Drop image" text) stays unchanged.

**Key invariants to preserve:**
- panX/panY stored as % of cell dimensions (unchanged)
- panScale range [1, 3] (unchanged, enforced in wheel handler)
- No changes to export.ts

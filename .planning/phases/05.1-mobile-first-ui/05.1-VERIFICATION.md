#### 1. Bottom Sheet Drag Gestures on Real Touch Device

**Test:** Open on iPhone or Android phone. Drag the sheet handle upward past 50px. Verify snap to half-state; drag again to full-state; drag down to collapse.
**Expected:** Smooth CSS transition; each snap state shows the correct sheet height; no page scroll interference during drag.
**Why human:** Touch pointer capture and passive event behavior differ between jsdom and mobile browsers.

#### 2. Canvas Fills 100vw on Phone (D-12)

**Test:** Open on a real phone at < 768px viewport. Verify the canvas occupies full screen width edge-to-edge with no horizontal scroll, side gap, or letterboxing.
**Expected:** Canvas touches both screen edges; bottom sheet collapsed handle sits below canvas.
**Why human:** CSS layout at device pixel ratios requires visual verification.

#### 3. Double-Tap Enters Pan Mode on Touch (D-11)

**Test:** On a phone, upload an image to a cell. Double-tap the cell. Verify pan mode yellow ring appears and sheet shows Exit Pan Mode button. Drag to reposition image. Pinch to zoom.
**Expected:** Pan mode activates; image repositions smoothly; pinch scales image within [1x, 3x]; Exit button exits pan mode.
**Why human:** `dblclick` simulation differs from real touch double-tap; passive event behavior requires real device.

#### 4. iOS File Picker Camera Option (D-15)

**Test:** On iPhone, tap an empty cell. Verify native iOS photo picker appears with both photo library and camera options.
**Expected:** iOS system sheet appears with "Photo Library" and "Camera" choices.
**Why human:** Requires iOS hardware; camera option display is OS-specific.

#### 5. Cell Swap by Touch Drag (D-16) — LIKELY BLOCKED

**Test:** On a phone, attempt to drag a filled cell over another cell to swap their images.
**Expected (per D-16):** Images swap when drag is released over another cell.
**Why human:** Native HTML5 drag events (`ondragstart`, `ondrop`) do not fire on iOS/Android touch. No @dnd-kit TouchSensor is configured. Cell swap may be desktop-only. This test is expected to FAIL on touch devices — result should inform whether D-16 needs a gap plan.

### Gaps Summary

**Gap 1 — Test regression in src/test/phase05-p03-responsive.test.tsx**

The plan updated `src/test/phase05-p03-responsive.test.tsx` to remove old desktop-notice assertions and add 5 new Phase 05.1 tests. However, the new tests fail with `ReferenceError: React is not defined` — this file (like other files in `src/test/`) lacks `import React from 'react'`. The SUMMARY noted 59 pre-existing failures in `src/test/`; the plan correctly added assertions to this file but inherited the broken infrastructure. The 27 new tests in `src/Editor/__tests__/` all pass correctly.

**Gap 2 — D-16 cell swap on mobile touch is not implemented**

Cell swap uses native HTML5 drag events (data transfer via `text/cell-id`). These events do not fire on iOS/Android touch without a polyfill or @dnd-kit TouchSensor. The CONTEXT.md states D-16 "ensure TouchSensor is included in DndContext sensor configuration" — but no DndContext with sensors exists anywhere in `src/`. The feature works on desktop (mouse) but not on touch. The VALIDATION.md correctly categorizes this as "Manual-Only" but the root cause is a missing implementation step, not just a test limitation.

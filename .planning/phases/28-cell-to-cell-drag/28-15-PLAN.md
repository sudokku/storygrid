Close 28-UAT Gap 2 — the ghost size too large on large source cells.

**The problem (28-UAT Test 1 + Test 4, raised TWICE):**

`DragPreviewPortal.tsx:41-43` renders the ghost `<img>` at `width: sourceRect.width, height: sourceRect.height` — uncapped. When the source cell is large (common on desktop with few cells, e.g. a 2-cell layout where each cell is ~400x800 viewport px), the ghost occupies most of the viewport during drag. Consequences:

1. User cannot see the target drop zones underneath the ghost while dragging.
2. User cannot see the accent-ring drag-over indicator (28-13 fix) on the hovered cell because the ghost occludes it.
3. The 5-zone overlay is invisible through the 0.8-opacity ghost.

User's exact words: "The floating copy of the image makes it hard to see what's behind it — let's make it 3/4 times smaller than original cell" (Test 1), and on Test 4: "Ghost size too large — enhancement request... user suggests capping with max-width + max-height preset so big cells don't have the ghost occluding drop zones."

**This is a SPEC CHANGE to GHOST-04, not a defect.** The original GHOST-04 spec said "ghost at source-cell size" — implemented correctly per spec. The spec itself is the pain point at large source sizes. The fix narrows the spec to "ghost at source-cell size, capped to GHOST_MAX_DIMENSION (200) on both axes, aspect ratio preserved."

**Why 200px:**

- User proposed "3/4 times smaller" — interpreted as "roughly 1/4 the source" (the user's phrasing was imprecise but the intent is clear: shrink by a factor of 3-4). For a typical large source (400x800 or 500x800), 1/4 is 100x200 or 125x200 — both fit comfortably under 200x200.
- User proposed "fixed preset" as the alternative — explicit max cap is the fixed-preset path.
- 200px is roughly 1/4 of a typical mobile-viewport-width cell (412px full viewport on iPhone 12 Pro) and ~1/4 of the canvas at 800px wide. It's visible enough to convey identity ("what am I dragging") but small enough that the target zones + accent ring are visible underneath.
- Named constant `GHOST_MAX_DIMENSION = 200` at the top of the file — trivial to retune if UAT still finds 200 too large (or too small).

**The fix:**

1. **Extract a pure helper** `computeCappedGhostSize(sourceRect: { width: number; height: number }, max: number): { width: number; height: number }`:

   ```typescript
   export function computeCappedGhostSize(
     sourceRect: { width: number; height: number },
     max: number,
   ): { width: number; height: number } {
     // Single uniform scale on both axes preserves the source's aspect ratio
     // exactly. scale = min(1, max/width, max/height):
     //   - if source fits inside the cap on both axes, scale = 1 → natural size.
     //   - otherwise scale = the smaller of max/width and max/height → the
     //     constraining axis hits the cap, the other axis shrinks by the same
     //     factor. Result: always within the cap, aspect ratio exact.
     const scale = Math.min(1, max / sourceRect.width, max / sourceRect.height);
     return {
       width: sourceRect.width * scale,
       height: sourceRect.height * scale,
     };
   }
   ```

   Exported so the test file can unit-test the math in isolation (same pattern as `computeDropZone` + `_testComputeZoneFromDragMove`).

2. **Consume the helper from both render branches** — the snapshot `<img>` and the empty-cell fallback `<div>`:

   ```typescript
   const { width, height } = computeCappedGhostSize(sourceRect, GHOST_MAX_DIMENSION);

   return (
     <DragOverlay adjustScale={false}>
       {active ? (
         ghostDataUrl ? (
           <img
             src={ghostDataUrl}
             style={{
               width,
               height,
               maxWidth: GHOST_MAX_DIMENSION,
               maxHeight: GHOST_MAX_DIMENSION,
               opacity: 0.8,
               display: 'block',
               objectFit: 'cover',
             }}
             alt=""
             draggable={false}
             data-testid="drag-ghost-img"
           />
         ) : (
           <div
             className="bg-[#1c1c1c]"
             style={{
               width,
               height,
               maxWidth: GHOST_MAX_DIMENSION,
               maxHeight: GHOST_MAX_DIMENSION,
               opacity: 0.8,
             }}
             data-testid="drag-ghost-fallback"
           />
         )
       ) : null}
     </DragOverlay>
   );
   ```

   The `maxWidth` / `maxHeight` CSS properties are a belt-and-suspenders ceiling: the computed width/height are already ≤ 200, but the CSS cap ensures the ghost never exceeds 200 even if `sourceRect` were mutated mid-render (defensive — in practice dragStore's sourceRect is immutable between beginCellDrag and end).

3. **Preserve `object-fit: cover`** on the `<img>` — already implicit (the original code did not set object-fit explicitly on the img style, relying on the natural `<img>` layout). Add `objectFit: 'cover'` explicitly so the aspect-ratio preservation is robust across browsers (Safari has historically had edge cases with `<img>` width + height overrides that violate intrinsic aspect). The snapshot is a full-rect capture so `cover` and `contain` produce identical results at matching aspect — `cover` is safe.

**Why this is strictly additive:**

- `width` + `height` values shrink in the cap case and stay unchanged in the non-cap case.
- `maxWidth` + `maxHeight` are NEW style properties — no existing property is removed.
- `objectFit: 'cover'` is NEW — same as above (no existing value to remove).
- The `<img>`'s `src`, `alt`, `draggable`, `data-testid`, `opacity`, `display` are UNCHANGED.
- The `<div>` fallback's `className`, `opacity`, `data-testid` are UNCHANGED.
- No new React imports, no new state, no new dragStore selectors.

/**
 * Ghost portal rendered into viewport space via @dnd-kit/core's DragOverlay
 * (REQ: GHOST-04, GHOST-05, GHOST-06).
 *
 * Renders the cell's canvas.toDataURL() snapshot as an <img> during drag.
 * Mounts outside the scaled canvas so there is no double-scale
 * (ARCHITECTURE.md §5; PITFALLS.md Pitfall 9 — never cloneNode on a canvas).
 *
 * GHOST-04 / GHOST-05: DragOverlay renders in viewport space via
 * createPortal(document.body) — outside the scaled canvas-surface. No scale
 * compensation is applied; dnd-kit's viewport-space transform is used as-is.
 * Any residual drift at non-1x canvasScale is absorbed by
 * MeasuringStrategy.Always on DndContext (gap-closure plan 28-12 — see
 * .planning/debug/touch-drag-unreliable.md for the misdiagnosis history).
 *
 * Gap-closure 28-15 (spec change): ghost dimensions are CAPPED at
 * GHOST_MAX_DIMENSION=200 on both axes with aspect ratio preserved via a
 * single uniform scale. This addresses the 28-UAT Gap 2 complaint
 * ("ghost too large — hides drop zones underneath on big cells"). See
 * .planning/phases/28-cell-to-cell-drag/28-UAT.md Gap 2 and the
 * ### Gap-Closure Plan 28-15 subsection in 28-VERIFICATION.md.
 *
 * Phase 27 ships this as a skeleton that renders null; Phase 28 wires it
 * to DragOverlay and the dragStore ghost-dataURL field.
 */

import { DragOverlay } from '@dnd-kit/core';
import { useDragStore } from './dragStore';

/**
 * Gap-closure 28-15: hard cap on ghost dimensions to prevent large source
 * cells from occluding drop zones + accent-ring indicators underneath the
 * ghost. 200px is roughly 1/4 of a typical mobile-viewport-width cell and
 * ~1/4 of a desktop canvas cell — visible enough to convey identity,
 * small enough to keep the target visuals visible. Retune here if UAT
 * re-confirmation prefers a different value.
 */
export const GHOST_MAX_DIMENSION = 200;

/**
 * Pure helper — exported for unit testing. Returns ghost dimensions capped
 * to `max` on both axes with aspect ratio preserved via a single uniform
 * scale factor.
 *
 * scale = min(1, max/width, max/height):
 *   - if both source axes fit inside the cap, scale = 1 → natural size.
 *   - otherwise scale = the smaller of max/width and max/height → the
 *     constraining axis hits the cap exactly, the other axis shrinks by
 *     the same factor. Result: always within the cap, aspect ratio exact.
 *
 * Examples (max=200):
 *   { width: 100, height: 150 } → { width: 100, height: 150 }        (scale=1)
 *   { width: 400, height: 800 } → { width: 100, height: 200 }        (scale=0.25)
 *   { width: 800, height: 100 } → { width: 200, height: 25 }         (scale=0.25)
 *   { width: 300, height: 300 } → { width: 200, height: 200 }        (scale=0.667)
 */
export function computeCappedGhostSize(
  sourceRect: { width: number; height: number },
  max: number,
): { width: number; height: number } {
  const scale = Math.min(1, max / sourceRect.width, max / sourceRect.height);
  return {
    width: sourceRect.width * scale,
    height: sourceRect.height * scale,
  };
}

export function DragPreviewPortal() {
  // Scoped primitive selectors — only re-render this portal when ghost state
  // changes (ARCHITECTURE.md §3). Avoid object-returning selectors that would
  // require useShallow.
  const status = useDragStore((s) => s.status);
  const ghostDataUrl = useDragStore((s) => s.ghostDataUrl);
  const sourceRect = useDragStore((s) => s.sourceRect);

  // Narrow sourceRect to non-null before destructuring width/height below.
  const active = status === 'dragging' && sourceRect !== null;

  // Gap-closure 28-15: compute capped dimensions once; reuse for both
  // branches (img for ghostDataUrl-present, div for D-10 empty-cell fallback).
  const capped = active
    ? computeCappedGhostSize(sourceRect, GHOST_MAX_DIMENSION)
    : null;

  return (
    <DragOverlay adjustScale={false}>
      {active && capped ? (
        ghostDataUrl ? (
          <img
            src={ghostDataUrl}
            style={{
              width: capped.width,
              height: capped.height,
              maxWidth: GHOST_MAX_DIMENSION,
              maxHeight: GHOST_MAX_DIMENSION,
              opacity: 0.8,
              display: 'block',
              objectFit: 'cover',
            }}
            alt=""
            draggable={false}
            data-testid="drag-ghost-img"
          />
        ) : (
          // D-10: empty-cell fallback — no canvas snapshot available, render a
          // dark div at source dims rather than a broken <img>. Cap applied
          // here too so large empty cells also get a sensible preview size.
          <div
            className="bg-[#1c1c1c]"
            style={{
              width: capped.width,
              height: capped.height,
              maxWidth: GHOST_MAX_DIMENSION,
              maxHeight: GHOST_MAX_DIMENSION,
              opacity: 0.8,
            }}
            data-testid="drag-ghost-fallback"
          />
        )
      ) : null}
    </DragOverlay>
  );
}

     * Pure helper — exported for unit testing. Returns ghost dimensions capped

     * scale factor: scale = min(1, max/width, max/height).
     *
     *   - if both source axes fit inside the cap, scale = 1 → natural size.
     *   - otherwise the constraining axis hits the cap exactly and the other
     *     axis shrinks by the same factor.
     *
     * Examples (max=200):

     *   { width: 400, height: 800 } → { width: 100, height: 200 }

    export function computeCappedGhostSize(
      sourceRect: { width: number; height: number },
      max: number,
    ): { width: number; height: number } {
      const scale = Math.min(1, max / sourceRect.width, max / sourceRect.height);
      return {
        width: sourceRect.width * scale,
        height: sourceRect.height * scale,
      };

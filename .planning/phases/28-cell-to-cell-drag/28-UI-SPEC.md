| Accent (10%) | `var(--primary)` | oklch(0.205 0 0) ≈ #1a1a1a | oklch(0.922 0 0) ≈ #e8e8e8 | See reserved-for list below |

**Accent reserved for:**
1. Drop target cell outline — 2px `outline` or `ring` using `var(--primary)` when pointer hovers any of a cell's 5 zones (DROP-04). Source: REQUIREMENTS.md DROP-04, CONTEXT.md D-02.
2. Drop-commit flash — 700ms accent-color flash on the landed cell after a successful drop (DROP-08). Source: REQUIREMENTS.md DROP-08.

**Drop zone icon colors (not accent — white-based):**
- Active zone icon: `text-white` at 100% opacity (`text-white/100`) + `scale-110` + `drop-shadow(0 0 6px rgba(255,255,255,0.8))`. Source: REQUIREMENTS.md DROP-02.
- Inactive zone icons: `text-white/30` (30% white), no scale, no glow. Source: REQUIREMENTS.md DROP-03.

**Ghost / source cell opacity:**
- Source cell during drag: `opacity: 0.4` (40%). Source: REQUIREMENTS.md GHOST-07 / CONTEXT.md DND locked list.
- Ghost (`DragOverlay` `<img>`): `opacity: 0.8` (80%). Source: REQUIREMENTS.md GHOST-03.

## Interaction Contract

This section captures the drag/drop interaction states. It supplements the standard template because this phase is interaction-heavy with no conventional form UI.

### Cursor States

| State | CSS | Source |
|-------|-----|--------|
| Leaf cell at rest | `cursor: grab` | REQUIREMENTS.md DRAG-01 |
| During active drag (document body) | `cursor: grabbing` on `document.body` | REQUIREMENTS.md DRAG-02 |

Implementation: add/remove `cursor-grabbing` class on `document.body` via `dragStore` status selector. Existing Phase 25 pattern uses the same approach (see LeafNode.tsx line 647).

### Drag Activation Thresholds

| Input | Constraint | Source |
|-------|-----------|--------|
| Mouse / pointer | 8px movement after pointerdown | REQUIREMENTS.md DRAG-04 |
| Touch | 250ms press-and-hold + 5px tolerance | REQUIREMENTS.md DRAG-03 |

Single `PointerSensor` handles both. Source: REQUIREMENTS.md DND-01, CONTEXT.md D decisions.

### Ghost Preview

| Property | Value | Source |
|----------|-------|--------|
| Generation method | `offscreen canvas → toDataURL()` on drag-start | REQUIREMENTS.md GHOST-01 |
| Rendered as | `<img>` inside `DragOverlay` portal | REQUIREMENTS.md GHOST-06 |
| Dimensions | Source cell dimensions (no cap) | REQUIREMENTS.md GHOST-04 |
| Opacity | 80% (`opacity: 0.8`) | REQUIREMENTS.md GHOST-03 |
| Content | Artwork only — no ActionBar, handles, selection ring | REQUIREMENTS.md GHOST-05 |
| Pointer tracking | Grab-point offset preserved via `@dnd-kit/modifiers` | REQUIREMENTS.md GHOST-02 |
| Scale | No scale transform on ghost at rest; `scale(1.04)` for lift effect | REQUIREMENTS.md DRAG-06 |

### Drop Zone Indicators (`DropZoneIndicators`)

5 zones tile the entire cell with no dead space (DROP-01, CANCEL-05):

| Zone | Position | Icon | CSS positioning |
|------|----------|------|-----------------|
| Center (swap) | `inset-[20%]` | `Maximize2` | Absolute, centered |
| Top edge (insert above) | `top-0 inset-x-0 h-[20%]` | `ArrowUp` | Absolute, flex-center |
| Bottom edge (insert below) | `bottom-0 inset-x-0 h-[20%]` | `ArrowDown` | Absolute, flex-center |
| Left edge (insert left) | `left-0 inset-y-0 w-[20%]` | `ArrowLeft` | Absolute, flex-center |
| Right edge (insert right) | `right-0 inset-y-0 w-[20%]` | `ArrowRight` | Absolute, flex-center |

Source: CONTEXT.md D-02, REQUIREMENTS.md DROP-01.

Overlay element rules:
- `pointer-events: none` — must not intercept drop events (ARCHITECTURE.md §6, PITFALLS.md).
- Rendered only when `dragStore.overId === cellId`.
- Zone icons use `size={32 / canvasScale}` to maintain physical stability at all zoom levels.

### Drop Target Cell Outline

When pointer is over any zone of a cell: `outline: 2px solid var(--primary)` or equivalent `ring-2 ring-primary ring-inset`. Applied to the cell's root wrapper. Removed on drag-end. Source: REQUIREMENTS.md DROP-04.

### Drop Commit Flash (DROP-08)

On successful drop: apply a CSS animation to the landed cell — accent-color flash that fades over 700ms. Implementation: add a class that triggers a `@keyframes` animation, remove class after 700ms via `setTimeout`. Color: `var(--primary)`. Duration: 700ms (Atlassian `largeDurationMs`). Source: REQUIREMENTS.md DROP-08.

### Drag-Start Wobble (DRAG-05)

Scale animation on drag activation: `1.00 → 1.05 → 1.02` over 150ms with `cubic-bezier(0.34, 1.56, 0.64, 1.0)`. Applied to the source cell element. Source: REQUIREMENTS.md DRAG-05.

Note: CONTEXT.md scopes DRAG-05 as out of scope for Phase 28 (deferred to Phase 29). Do not implement the wobble in this phase.

### Cancel — Out-of-Canvas Release (CANCEL-03)

Release outside `GridCanvas` → cancel with no `moveCell` call. Source: REQUIREMENTS.md CANCEL-03, CONTEXT.md locked requirements.

### Cancel — Origin Cell Release (CANCEL-04)

Release on origin cell → no-op. No undo-history entry. Source: REQUIREMENTS.md CANCEL-04.

Note: CANCEL-01 (ESC key) and CANCEL-02 (snap-back animation) are deferred to Phase 29. Do not implement them in Phase 28.

### `data-dnd-ignore` Guard

Add `data-dnd-ignore="true"` to `Divider.tsx` root and `OverlayLayer.tsx` root. Prevents PointerSensor from consuming drag events on these interactive overlays. Source: CONTEXT.md Claude's Discretion.

### Cross-Device CSS (CROSS-02 through CROSS-05)

| Property | Value | Element |
|----------|-------|---------|
| `touch-action: none` | While drag available | Draggable leaf cells |
| `-webkit-touch-callout: none` | Always | Draggable leaf cells |
| `user-select: none` | During active drag | `document.body` |
| `contextmenu` `preventDefault` | During active drag | `document` listener |

Note: CROSS-02 through CROSS-05 are listed in REQUIREMENTS.md but CONTEXT.md domain block explicitly defers "Mobile haptics, MobileSheet auto-collapse, cross-device CSS (touch-action, -webkit-touch-callout, user-select, contextmenu)" to Phase 30. Do not implement CROSS-02 through CROSS-05 in Phase 28 — Phase 30 owns them. CROSS-01 (single pointer-event stream) is covered by the `PointerSensor` choice.

## Animation Contract

| Interaction | Duration | Easing | Source |
|-------------|----------|--------|--------|
| Drop commit flash (DROP-08) | 700ms | ease-out (fade-out) | REQUIREMENTS.md DROP-08 |
| Source cell dim on drag-start | 150ms | ease-out | UI-INTELLIGENCE.md "150–300ms micro-interactions" |
| Ghost opacity settle | 0ms (set immediately) | — | REQUIREMENTS.md GHOST-03 |
| Drop zone icon appear | 150ms | ease-out | UI-INTELLIGENCE.md default |
| Zone active scale (1.1) | 100ms | ease-out | UI-INTELLIGENCE.md |

Respect `prefers-reduced-motion`: disable the 700ms flash animation and use an immediate 1-frame opacity change for source cell dim. Source: UI-INTELLIGENCE.md HIGH rule.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1c1c1c` | Sidebar background, panel background |
| Secondary (30%) | `#2a2a2a` | Chip background, button resting state, input background, dividers, borders |
| Accent (10%) | `#3b82f6` (blue-500) | Active preset chip highlight, selected toggle segment, slider `accent-color` |

Second semantic color (destructive): `text-red-400` + `hover:bg-red-500/20` — "Reset cell" button only.

Source: Sidebar.tsx established dark palette — `bg-[#1c1c1c]`, `bg-[#2a2a2a]`, `bg-[#3b82f6]`, `text-neutral-300/400/500`.

## Interaction Contract

### Preset Chip Selection (D-08, D-11)
- Tap/click a chip → calls `applyPreset(nodeId, presetName)` → loads preset values into all 4 sliders + sets `effects.preset = name` → single history snapshot
- Active chip shows `ring-2 ring-[#3b82f6]` on thumbnail; all other chips show no ring
- Chips render with static pre-rendered sample thumbnails (not live per-cell renders — D-09)
- Horizontal scroll on overflow — no pagination, no arrows

### Slider Drag (D-13, D-14)
- `pointerdown` on slider → `setPointerCapture` on the row div
- `pointermove` → calls `setEffects(nodeId, { [field]: newValue })` — no history commit, live WYSIWYG update
- `pointerup` → calls `commitEffects(nodeId, finalEffects)` — exactly one `pushSnapshot`
- Moving any slider while a preset is active → `effects.preset = null` on first `pointermove` (D-15) — chip ring clears immediately
- Keyboard accessibility: native `<input type="range">` handles arrow key step (step=1 for brightness/contrast/saturation, step=1 for blur)

### Reset Effects (D-16, D-17)
- Click "Reset effects" → calls `resetEffects(nodeId)` → sets `preset: null`, `brightness: 0`, `contrast: 0`, `saturation: 0`, `blur: 0` → one `pushSnapshot`
- Chip ring clears. Sliders all return to center/zero.

### Reset Cell (D-16, D-17)
- Click "Reset cell" → calls `resetCell(nodeId)` → clears effects AND pan/zoom/fit/backgroundColor → one `pushSnapshot`
- No confirmation dialog — undo available via Ctrl+Z

### Effects Panel Visibility
- Effects section renders only when a leaf node is selected (inside SelectedCellPanel)
- If the selected leaf has no media, show the disabled copy: `Add media to apply effects` in `text-xs text-neutral-500` — all sliders and chips are `opacity-40 pointer-events-none`

### WYSIWYG Guarantee
- Effects are applied exclusively inside `drawLeafToCanvas()` via `ctx.filter` — never via CSS `filter:` on the canvas DOM element (D-06)
- Preview canvas and PNG/MP4 export share the same draw path — WYSIWYG by construction

## Preset Value Reference

Canonical values live in `src/lib/effects.ts` as `PRESET_VALUES`. UI-SPEC records the approved starting values for visual design alignment:

| Preset | Brightness | Contrast | Saturation | Blur |
|--------|-----------|---------|-----------|------|
| B&W | 0 | 0 | -100 | 0 |
| Sepia | +5 | +10 | -80 | 0 |
| Vivid | 0 | +15 | +40 | 0 |
| Fade | +10 | -20 | -15 | 0 |
| Warm | +5 | 0 | +10 | 0 |
| Cool | -5 | 0 | +10 | 0 |

Note: Sepia uses `saturation: -80` (not -100) to preserve a warm undertone. Planner may tune these values during implementation and should test against sample portrait photos.

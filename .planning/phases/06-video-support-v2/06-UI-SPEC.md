## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, scrubber track padding |
| sm | 8px | Compact element spacing, timeline control gaps |
| md | 16px | Default element spacing, timeline bar horizontal padding |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Play/pause button touch target: 44px minimum (mobile touch standard; D-09 control)
- Timeline bar height: 48px total (12px scrubber track + padding; fits one row of controls)
- Scrubber range input height: 4px track, 16px thumb

Source: 8-point grid default; touch target exception is standard mobile accessibility practice.

## Component Inventory

New components introduced in this phase:

### PlaybackTimeline

Location: `src/Editor/PlaybackTimeline.tsx`

- Container: `h-12` (48px) horizontal flex row, `bg-card` background (`oklch(0.205 0 0)`), `border-t border-border`, `px-4 gap-3`, full width of canvas column
- Visibility: rendered only when `hasVideoCell(root, mediaRegistry)` is true (D-10). Mounted/unmounted reactively.
- Play/Pause button: 44px touch target (`w-11 h-11`), icon-only, `text-foreground`, hover `bg-muted`, icon `Play` / `Pause` from lucide at 16px. When playing, icon switches to Pause. No explicit accent color on button face; accent is reserved for the scrubber.
- Scrubber: `<input type="range">` full width between play button and time display. Track: 4px height, `bg-muted` background. Filled track left of thumb: accent blue via CSS `::-webkit-slider-runnable-track` styled portion. Thumb: 16px circle, accent blue (`#3b82f6` fallback).
- Time display: `0:03 / 0:08` format, `text-muted-foreground`, 12px, `tabular-nums`, min-width to avoid layout shift on changing values.

### Toast (extended)

Existing `src/Editor/Toast.tsx` extended with two new states:

- `loading-ffmpeg`: spinner + "Loading ffmpeg..." (Loader2 icon, same container style as 'preparing')
- `encoding`: spinner + "Encoding 42%..." with dynamic percentage (Loader2 icon, same container style as 'exporting')

Toast container style unchanged: `bg-[#1c1c1c] border border-white/10 rounded-md px-3 py-2` (existing).

### ExportSplitButton (modified)

No new visual elements. Behavioral change: when video cells exist, the popover hides PNG/JPEG format selector and quality slider (D-14). The main button label stays "Export". Format/quality controls hidden via conditional render, not disabled.

## Interaction States

### Video Cell (LeafNode with video mediaId)

| State | Visual |
|-------|--------|
| Loading (video.readyState < 2) | Same empty-state dashed border with Loader2 spinner at center, 16px, `text-muted-foreground` |
| Paused | Still frame drawn on canvas; no play indicator overlay |
| Playing | Canvas updates via rAF; no overlay added |
| Pan mode | Same behavior as image cells; pan/zoom applies in canvas draw |

Note: No play-button overlay on the cell itself. Playback is controlled exclusively via the timeline bar (D-09).

### Timeline Bar States

| State | Visual |
|-------|--------|
| No video cells | Not rendered (D-10) |
| Paused | Play icon shown; scrubber at current position; time display static |
| Playing | Pause icon shown; scrubber and time display update in real time |
| Seeking (drag scrubber) | All video refs seek immediately; canvas redraws still frame during seek drag |

### Export Button States

| State | Visual |
|-------|--------|
| Image-only grid | Existing behavior: popover shows PNG/JPEG format + quality |
| Video cells present | Popover hides format/quality controls; shows "Exports as MP4 (H.264)" label instead |
| ffmpeg loading | Toast: Loader2 + "Loading ffmpeg..." |
| Encoding in progress | Toast: Loader2 + "Encoding XX%..." (updates with percentage) |
| Encode complete | Toast dismissed; download triggered automatically |
| Error | Toast: AlertCircle + "Export failed." + "Try again" button |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Export" (unchanged — same button; format auto-detected) |

Destructive actions in this phase: none. Video blob URLs are cleared on reload silently (D-03/D-04 — no confirmation needed by design).

## Layout Contract

### Timeline Bar Position

```
┌─────────────────────────────────────┐
│  Toolbar (full width)               │
├─────────────┬───────────────────────┤
│             │  CanvasArea           │
│  Sidebar    │  (canvas column)      │
│  (desktop)  ├───────────────────────┤
│             │  PlaybackTimeline     │  ← this phase
│             │  (canvas column only) │
└─────────────┴───────────────────────┘
```

- Timeline sits inside the same flex column as CanvasArea, below it
- Timeline is NOT full-width — it is scoped to the canvas column only (D-08)
- On mobile: timeline is inside the canvas column (above the mobile sheet snap zone), not inside MobileSheet

### Mobile Consideration

On mobile (`< 768px`), the PlaybackTimeline renders in the canvas column above the bottom sheet handle area. It must not overlap the MobileSheet drag handle (32px dead zone at bottom of canvas column). Timeline height 48px + 8px gap from canvas bottom edge avoids the conflict.

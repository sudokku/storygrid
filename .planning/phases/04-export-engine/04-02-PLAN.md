    2. Create `src/Editor/ExportSplitButton.tsx` — the split button with popover per D-01 through D-05, D-08, D-09:

       **Props:**
       ```typescript
       interface ExportSplitButtonProps {
         exportRef: React.RefObject<HTMLDivElement | null>;
       }
       ```

       **Internal state:**
       - `popoverOpen: boolean` (local useState, default false)
       - `toastState: ToastState` (local useState, default null)
       - `popoverRef: React.RefObject<HTMLDivElement>` for outside click detection
       - `containerRef: React.RefObject<HTMLDivElement>` for outside click exclusion (the whole split button)

       **Store connections:**
       - `isExporting` from `useEditorStore`
       - `exportFormat` from `useEditorStore`
       - `exportQuality` from `useEditorStore`
       - `setIsExporting` from `useEditorStore`
       - `setExportFormat` from `useEditorStore`
       - `setExportQuality` from `useEditorStore`
       - `root` from `useGridStore`
       - `mediaRegistry` from `useGridStore`

       **handleExport function (async):**
       ```
       1. If isExporting, return early
       2. Check hasVideoCell(root, mediaRegistry) — if true, set toastState='video-blocked' per D-09, return
       3. If exportRef.current is null, return
       4. setIsExporting(true)
       5. Close popover if open
       6. try {
            const dataUrl = await exportGrid(exportRef.current, exportFormat, exportQuality, (stage) => setToastState(stage));
            setToastState(null); // dismiss toast on success per D-07
            const ext = exportFormat === 'jpeg' ? 'jpg' : 'png';
            const filename = `storygrid-${Date.now()}.${ext}`;  // per D-16, D-17
            downloadDataUrl(dataUrl, filename);
          } catch (err) {
            setToastState('error');
          } finally {
            setIsExporting(false);  // per RESEARCH Pitfall 7 — MUST be in finally
          }
       ```

       **Outside click handler per RESEARCH Pitfall 6:**
       - useEffect that adds `mousedown` listener on `document` when `popoverOpen === true`
       - Handler checks if click target is inside `popoverRef.current` or `containerRef.current` — if not, close popover
       - Also close on Escape key (`keydown` listener)
       - Cleanup: remove listeners on close or unmount

       **Split button structure per UI-SPEC:**
       - Shared container: `<div ref={containerRef} className="relative flex items-center border border-white/10 rounded">` (relative for popover positioning)
       - Left segment: `<button onClick={handleExport} disabled={isExporting} aria-label="Export PNG" className="flex items-center gap-2 h-8 px-2 rounded-l hover:bg-white/10 transition-colors text-neutral-300 disabled:opacity-40 disabled:cursor-not-allowed">`
         - Content: `<Download size={16} />` + `<span className="text-xs">Export {exportFormat === 'jpeg' ? 'JPEG' : 'PNG'}</span>`
         - aria-label updates dynamically: `Export ${exportFormat.toUpperCase()}`
       - Divider: `<div className="w-px h-4 bg-white/20 self-center" />`
       - Right segment: `<button onClick={() => !isExporting && setPopoverOpen(!popoverOpen)} disabled={isExporting} aria-label="Export settings" aria-expanded={popoverOpen} className="flex items-center justify-center w-5 h-8 rounded-r hover:bg-white/10 transition-colors text-neutral-400 disabled:opacity-40 disabled:cursor-not-allowed ${popoverOpen ? 'bg-white/10' : ''}">`
         - Content: `<ChevronDown size={12} />`

       **Popover per UI-SPEC:**
       - Render conditionally when `popoverOpen === true`
       - Container: `<div ref={popoverRef} className="absolute top-full right-0 mt-1 w-56 bg-[#1c1c1c] border border-white/10 rounded-md shadow-lg p-2 z-50" role="dialog" aria-label="Export settings">`
       - Format toggle label: `<span className="text-xs text-neutral-400 mb-2 block">Format</span>`
       - Format toggle container: `<div className="flex rounded border border-white/10 overflow-hidden" role="radiogroup">`
         - PNG button: `<button onClick={() => setExportFormat('png')} className="flex-1 h-7 text-xs rounded transition-colors ${exportFormat === 'png' ? 'bg-white/15 text-white' : 'text-neutral-400 hover:text-neutral-300'}" role="radio" aria-checked={exportFormat === 'png'}>PNG</button>`
         - JPEG button: same pattern with `'jpeg'`
       - Quality slider (visible only when JPEG): wrap in `<div className={exportFormat === 'jpeg' ? 'mt-3' : 'hidden'}>`:
         - Label: `<span className="text-xs text-neutral-400">Quality: {Math.round(exportQuality * 100)}%</span>`
         - Input: `<input type="range" min={0.7} max={1.0} step={0.05} value={exportQuality} onChange={(e) => setExportQuality(parseFloat(e.target.value))} className="w-full accent-white mt-2" aria-label="Export quality" aria-valuetext={`${Math.round(exportQuality * 100)} percent`} />`
       - Download button: `<button onClick={handleExport} disabled={isExporting} className="w-full h-8 mt-3 rounded text-xs font-medium bg-white/10 hover:bg-white/15 text-white transition-colors disabled:opacity-40 disabled:cursor-not-allowed">Download {exportFormat === 'jpeg' ? 'JPEG' : 'PNG'}</button>`

       **Toast rendering:** Render `<Toast state={toastState} onRetry={handleExport} onDismiss={() => setToastState(null)} />` as the last element in the component return (inside a React Fragment or after the split button div).

       Import `Download, ChevronDown` from lucide-react.

    **handleRetry:** The `onRetry` callback passed to Toast is the same `handleExport` function — clicking "Try again" re-triggers the export with the same settings per CONTEXT.md error recovery pattern.

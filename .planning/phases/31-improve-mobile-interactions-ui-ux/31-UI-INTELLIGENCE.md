## Touch Interaction & Accessibility

### Touch Target Size (High)
- **Do:** Minimum 44√ó44px touch targets ‚Äî `min-h-[44px] min-w-[44px]`
- **Don't:** Tiny `w-6 h-6` clickable areas on mobile

### Touch Spacing (Medium)
- **Do:** Minimum 8px gap between touch targets ‚Äî `gap-2 between buttons`
- **Don't:** `gap-0` or `gap-1` between tightly packed elements

### Touch-Friendly (High)
- **Do:** Increase touch targets on mobile
- **Don't:** Same tiny buttons on mobile as desktop

### Haptic Feedback (Low)
- **Do:** Use for confirmations and important actions ‚Äî `navigator.vibrate(10)`
- **Don't:** Overuse vibration

### Focus States (High)
- **Do:** Visible focus rings ‚Äî `focus:ring-2 focus:ring-blue-500`
- **Don't:** `outline-none` without replacement

## Animation & Motion

### Duration Timing (Medium)
- **Do:** 150‚Äì300ms for micro-interactions ‚Äî `transition-all duration-200`
- **Don't:** Animations longer than 500ms for UI transitions

### Easing Functions (Low)
- **Do:** ease-out for entering, ease-in for exiting
- **Don't:** linear easing for UI transitions

### Reduced Motion (High)
- **Do:** Check `prefers-reduced-motion` media query
- **Don't:** Ignore accessibility motion settings

### Hover vs Tap (High)
- **Do:** Use click/tap for primary interactions
- **Don't:** Rely only on `onMouseEnter` for important actions

## Viewport & Layout

### Viewport Meta (High)
- **Do:** `width=device-width, initial-scale=1` ‚Äî apply `maximum-scale=1, user-scalable=no` for single-screen canvas apps
- **Don't:** Missing or incorrect viewport meta

### Viewport Units (Medium)
- **Do:** Use `dvh` or account for mobile browser chrome
- **Don't:** Use `100vh` for full-screen mobile layouts ‚Äî use `min-h-dvh` or `min-h-screen`

### Horizontal Scroll (High)
- **Do:** Ensure content fits viewport width ‚Äî `max-w-full overflow-x-hidden`
- **Don't:** Content wider than viewport

## Navigation & Sheet Dismissal

### Back Button (High)
- **Do:** Preserve navigation history with `history.pushState()`
- **Don't:** Break browser/app back button behavior with `location.replace()`

### Active State (Medium)
- **Do:** Highlight active nav item with color/underline
- **Don't:** All links same style

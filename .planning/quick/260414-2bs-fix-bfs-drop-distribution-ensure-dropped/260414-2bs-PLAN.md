# Quick Task 260414-2bs: Fix BFS Drop Distribution

**Date:** 2026-04-13
**Status:** Complete

## Goal

Fix `autoFillCells` so that dropping N files always produces a balanced tree where
all leaf nodes are at the same depth (true BFS), instead of the current DFS-like chain.

## Root Cause

`autoFillCells` used `lastFilledNodeId` to track which leaf to split on overflow.
This always selected the most recently filled leaf, causing each successive split to
go one level deeper on the same branch — creating a right-heavy linear chain.

## Tasks

### Task 1: Export `getParentDirection` from tree.ts
- **File:** `src/lib/tree.ts`
- **Action:** Add and export `getParentDirection(root, nodeId)` — returns the parent
  container's direction or null when the node is the root.
- **Verify:** Imported successfully by media.ts.

### Task 2: Rewrite `autoFillCells` with two-phase BFS approach
- **File:** `src/lib/media.ts`
- **Action:** Replace last-filled-tracking with:
  - **Phase 1 (Pre-expand):** FIFO queue seeded with current BFS leaves. Dequeue
    oldest leaf, split it using the cross-direction of its parent (forces Case C —
    never Case B sibling-append at same depth). Enqueue original + new sibling at
    back. Stop when empty-leaf count ≥ N.
  - **Phase 2 (Fill):** Fill empty leaves in BFS order.
- **Verify:** 8-file drop produces 4 groups × 2 leaves at the same depth.

### Task 3: Update tests
- **File:** `src/test/media.test.ts`
- **Action:** Update the two overflow-split tests to match new behaviour:
  - Rename "auto-splits last filled leaf" → "pre-expands tree with 1 BFS split"
  - Update direction assertions: H, V, H, V → H, V, V, H (cross-parent-direction)
